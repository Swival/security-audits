# PoC 058 — `nghttp2_session_consume()` errors silently dropped

`h2_stream_in_consumed()` calls `nghttp2_session_consume()` but discards
the return value and decrements `consumed` regardless. A libnghttp2 failure
goes unobserved; the stream's beam reports the bytes as accounted, while
HTTP/2 flow-control state may not have been updated.

## Setup

Build with `mod_http2` and a faulted libnghttp2 (LD_PRELOAD shim is the
simplest way to inject the failure deterministically).

`fault_nghttp2.c`:

```c
#define _GNU_SOURCE
#include <dlfcn.h>
#include <stdio.h>
#include <string.h>
#include <stdlib.h>

typedef int (*real_consume_t)(void *, int, size_t);
static real_consume_t real_consume;
static int counter;

int nghttp2_session_consume(void *s, int sid, size_t len) {
    if (!real_consume)
        real_consume = (real_consume_t)dlsym(RTLD_NEXT,
                                              "nghttp2_session_consume");
    if (++counter > 5) {
        fprintf(stderr, "[shim] forcing NGHTTP2_ERR_NOMEM on consume\n");
        return -901; /* NGHTTP2_ERR_NOMEM */
    }
    return real_consume(s, sid, len);
}
```

```
gcc -shared -fPIC fault_nghttp2.c -o /tmp/fault_nghttp2.so -ldl
```

`httpd.conf`:

```
Listen 127.0.0.1:8443
LoadModule ssl_module    modules/mod_ssl.so
LoadModule http2_module  modules/mod_http2.so
ServerName localhost
SSLEngine on
SSLCertificateFile    server.crt
SSLCertificateKeyFile server.key
Protocols h2 http/1.1
H2WindowSize 65535
H2StreamMaxMemSize 65536
DocumentRoot /var/www
<Location /upload>
    SetHandler dump
</Location>
```

A small upload sink (`/var/www/upload.cgi` or any handler) is fine; the
key requirement is that the request body is consumed by Apache so the
beam callback fires.

## Trigger

Run httpd with the shim preloaded and push a sustained body upload:

```
LD_PRELOAD=/tmp/fault_nghttp2.so httpd -X &

# h2load uploading 4 MiB across multiple streams
h2load -d /tmp/4mb.bin -n 16 -c 1 -m 4 \
       https://127.0.0.1:8443/upload
```

## Observed effect

Pre-patch: `error_log` shows the shim's diagnostic
`[shim] forcing NGHTTP2_ERR_NOMEM on consume`, but no APR-level error is
logged from `h2_stream_in_consumed()`. Beam accounting in
`h2_bucket_beam.c:128` advances `recv_bytes_reported` regardless. The
session keeps running until the next libnghttp2-internal check fires;
downstream symptoms include stalled uploads or `WINDOW_UPDATE` frames not
being emitted (visible with `h2load -v` or Wireshark capture on
`tcp.port == 8443` after TLS keylog decryption).

Post-patch the same trigger causes the stream to abort with
`APR_EGENERAL`, and `error_log` records the propagated failure at the
`h2_stream` level immediately after the shim's diagnostic.

## Log marker

```
[shim] forcing NGHTTP2_ERR_NOMEM on consume       # pre- and post-patch
[http2:error] ... h2_stream_in_consumed: ...      # post-patch only
```

The presence of the shim line without the matching httpd-side error is
the observable signal that the libnghttp2 failure was discarded.
