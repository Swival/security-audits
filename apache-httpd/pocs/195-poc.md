# PoC 195 - Signed Overflow Parsing Initial Burst

`mod_ratelimit` parses `rate-initial-burst` through `atoi()` and multiplies by
1024 in signed `int`, so a value of `2097152` overflows before the
`burst <= 0` validation can reject it.

## Setup

Same build as PoC 194: httpd with `mod_ratelimit`, ideally instrumented with
`-fsanitize=undefined`.

`/tmp/httpd.conf`:

```apache
ServerName 127.0.0.1
Listen 127.0.0.1:8080
LoadModule mpm_event_module     modules/mod_mpm_event.so
LoadModule unixd_module         modules/mod_unixd.so
LoadModule authz_core_module    modules/mod_authz_core.so
LoadModule env_module           modules/mod_env.so
LoadModule ratelimit_module     modules/mod_ratelimit.so
LoadModule dir_module           modules/mod_dir.so
LoadModule mime_module          modules/mod_mime.so

DocumentRoot "/tmp/htdocs"

<Location "/burst">
    SetEnv rate-limit         128
    SetEnv rate-initial-burst 2097152
    SetOutputFilter RATE_LIMIT
</Location>
<Directory "/tmp/htdocs">
    Require all granted
</Directory>
```

```sh
mkdir -p /tmp/htdocs
python3 -c "open('/tmp/htdocs/index.html','wb').write(b'B'*8192)"
export UBSAN_OPTIONS=print_stacktrace=1:halt_on_error=1
httpd -DFOREGROUND -f /tmp/httpd.conf &
```

`rate-limit` is kept small and valid so execution proceeds past PoC 194's
guard and reaches the `rate-initial-burst` parsing block.

## Trigger

```sh
curl -sv http://127.0.0.1:8080/burst/index.html >/dev/null
```

## Observed effect

- UBSan-instrumented build aborts inside `rate_limit_filter()` with:

  ```
  modules/filters/mod_ratelimit.c:99:24: runtime error: signed integer overflow:
      2097152 * 1024 cannot be represented in type 'int'
  ```

- Without UBSan, `burst` wraps to `INT_MIN` (`-2147483648`). The
  `burst <= 0` arm fires and `error.log` records:

  ```
  AH03489: rl: disabling burst: rate-initial-burst = 2097152 (too high?)
  ```

  but the wrap has already invoked C undefined behavior, and on
  `-O2 -fno-strict-overflow` mismatched builds the burst may instead be
  applied as the wrapped negative value, yielding immediate filter stalls.
- Log marker: child SIGABRT under UBSan, or AH03489 disable message
  without it.

Standalone confirmation without httpd:

```sh
cat > /tmp/burst.c <<'EOF'
#include <stdio.h>
#include <stdlib.h>
int main(void){ int b = atoi("2097152") * 1024; printf("b=%d\n", b); }
EOF
cc -O0 -fsanitize=undefined /tmp/burst.c -o /tmp/burst && /tmp/burst
```
