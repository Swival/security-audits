# PoC 151 — `h2_proxy_fifo_term()` Does Not Wake Blocked Waiters

`h2_proxy_fifo_term()` sets `aborted = 1` but never broadcasts the FIFO
condition variables, so threads blocked in `pull` or `push` sleep
forever.

## Setup

This is a unit-style PoC against `h2_proxy_util.c`, exercising the
public FIFO API directly. Build a small driver linked against APR and the
extracted `h2_proxy_fifo` translation unit.

`Makefile`:

```make
APR_CFLAGS != apr-1-config --cflags --cppflags --includes
APR_LIBS   != apr-1-config --link-ld --libs

poc151: poc151.c
	$(CC) $(APR_CFLAGS) -I/path/to/httpd-2.4.66/modules/http2 \
	      poc151.c /path/to/httpd-2.4.66/modules/http2/h2_proxy_util.c \
	      $(APR_LIBS) -lpthread -o poc151
```

`poc151.c`:

```c
#include <stdio.h>
#include <apr_thread_proc.h>
#include "h2_proxy_util.h"

static void *APR_THREAD_FUNC waiter(apr_thread_t *t, void *arg)
{
    h2_proxy_fifo *f = arg;
    void *e;
    apr_status_t rv = h2_proxy_fifo_pull(f, &e);
    fprintf(stderr, "puller returned %d\n", rv);
    apr_thread_exit(t, 0);
    return NULL;
}

int main(void)
{
    apr_pool_t *p; apr_thread_t *th;
    apr_initialize(); apr_pool_create(&p, NULL);
    h2_proxy_fifo *f;
    h2_proxy_fifo_create(&f, p, 4);
    apr_thread_create(&th, NULL, waiter, f, p);
    apr_sleep(apr_time_from_sec(1));      /* puller now blocked */
    h2_proxy_fifo_term(f);                /* should wake it; doesn't */
    apr_status_t rv;
    apr_thread_join(&rv, th);             /* hangs forever */
    return 0;
}
```

## Trigger

```bash
make poc151
timeout 5 ./poc151 ; echo "exit=$?"
```

## Observed effect

* `puller returned …` line never prints.
* `timeout` kills the process after five seconds: `exit=124`.
* Switching `h2_proxy_fifo_term` to also broadcast `not_empty`/`not_full`
  (per the patch) makes the puller return `APR_EOF` immediately.
* The same construction with a full FIFO and a blocked
  `h2_proxy_fifo_push` reproduces the symmetric hang on `not_full`.

Confirms the missing broadcast at `modules/http2/h2_proxy_util.c:1169`.
