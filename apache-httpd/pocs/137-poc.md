# PoC 137 - AuthBasicFake leaks generated Authorization header to error log

`mod_auth_basic` logs the full generated `Authorization: Basic <base64>`
header at info level, exposing reversible credential material to anyone
with read access to the error log.

## Setup

Lab httpd 2.4.66 with `mod_auth_basic`, a backend that requires Basic auth,
and an `AuthBasicFake` directive supplying the credentials.

`conf/httpd.conf`:

```
ServerName localhost
Listen 127.0.0.1:8080
LoadModule mpm_prefork_module    modules/mod_mpm_prefork.so
LoadModule authn_core_module     modules/mod_authn_core.so
LoadModule authz_core_module     modules/mod_authz_core.so
LoadModule auth_basic_module     modules/mod_auth_basic.so
LoadModule unixd_module          modules/mod_unixd.so
LoadModule log_config_module     modules/mod_log_config.so

ErrorLog logs/error_log
LogLevel auth_basic:info

DocumentRoot "/tmp/htdocs"
<Directory "/tmp/htdocs">
    AuthType Basic
    AuthName "fake"
    AuthBasicProvider anon
    Anonymous *
    Anonymous_NoUserID On
    AuthBasicFake backend-svc s3cret-backend-pass
    Require all granted
</Directory>
```

`Anonymous *` keeps the request authenticated without a real user file, so
the fake-auth fixup runs and writes the `Authorization` header.

## Trigger

```
curl -s http://127.0.0.1:8080/ >/dev/null
grep AuthBasicFake logs/error_log
```

## Observed effect

`logs/error_log` records (line numbers truncated):

```
[auth_basic:info] [pid 12345] [client 127.0.0.1:54321] AH02457:
  AuthBasicFake: "Authorization: Basic YmFja2VuZC1zdmM6czNjcmV0LWJhY2tlbmQtcGFzcw=="
```

The base64 string is reversible:

```
$ printf 'YmFja2VuZC1zdmM6czNjcmV0LWJhY2tlbmQtcGFzcw==' | base64 -d
backend-svc:s3cret-backend-pass
```

Anyone with read access to `error_log` (operators, log shippers, SIEM
indexes, log rotation backups) recovers the configured fake credentials.
With `LogLevel info` enabled by default in many production deployments the
header is captured on every request that hits a fake-auth fixup.

The patch replaces the value-bearing message with
`AuthBasicFake: Authorization header set`, preserving the operational
breadcrumb without leaking the credential.
