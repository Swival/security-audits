# PoC 106 — Unique ID Counter Wraps Early

`mod_unique_id` reduces its 32-bit counter with `% APR_UINT16_MAX` (65535)
instead of `% 65536`, so only 65,535 distinct counter values are emitted and
duplicate `UNIQUE_ID` values appear when a single child handles 65,536
requests in one second.

## Setup

The duplicate pattern is independent of the network and can be reproduced
deterministically with a tiny standalone driver that mirrors the encoded
counter byte. No live server is required.

`/tmp/uniqueid_demo.c`:

```c
#include <stdio.h>
#include <stdint.h>
#include <arpa/inet.h>

#define APR_UINT16_MAX 0xffff

int main(void) {
    uint32_t a = 12345;
    uint32_t b = a + 65535;
    uint16_t ca_bug = htons(a % APR_UINT16_MAX);
    uint16_t cb_bug = htons(b % APR_UINT16_MAX);
    uint16_t ca_fix = htons(a % (APR_UINT16_MAX + 1));
    uint16_t cb_fix = htons(b % (APR_UINT16_MAX + 1));

    printf("buggy:  %u and %u -> %04x %04x (collision=%d)\n",
           a, b, ca_bug, cb_bug, ca_bug == cb_bug);
    printf("fixed:  %u and %u -> %04x %04x (collision=%d)\n",
           a, b, ca_fix, cb_fix, ca_fix == cb_fix);
    return 0;
}
```

## Trigger

```bash
cc -O2 -o /tmp/uniqueid_demo /tmp/uniqueid_demo.c
/tmp/uniqueid_demo
```

## Observed effect

```text
buggy:  12345 and 78880 -> 3930 3930 (collision=1)
fixed:  12345 and 78880 -> 3930 3931 (collision=0)
```

The buggy reduction collides; the patched modulus does not.

## End-to-end reproduction sketch

To observe the same collision in an actual server log, the per-child counter
must reach 65,536 within a single second:

```apache
LoadModule unique_id_module modules/mod_unique_id.so
LogFormat "%{UNIQUE_ID}e" uniq
CustomLog logs/uniq_log uniq
StartServers 1
ServerLimit 1
ThreadLimit 1
ThreadsPerChild 1
MaxRequestsPerChild 0
```

Drive a single keepalive connection (so all requests stay on one child/thread
within one timestamp second):

```bash
python3 -c '
import socket, sys
s = socket.create_connection(("127.0.0.1", 8080))
req = b"GET / HTTP/1.1\r\nHost: x\r\nConnection: keep-alive\r\n\r\n"
for _ in range(65536):
    s.sendall(req)
s.shutdown(socket.SHUT_WR)
while s.recv(65536):
    pass
'

sort logs/uniq_log | uniq -d | head
```

With the buggy modulus, `uniq -d` prints a duplicate `UNIQUE_ID`; with the
patch the entire log is unique.

## Marker

Duplicate line on stdout from `uniq -d`, or `collision=1` from the standalone
demo.
