# 033 - mod_cgid sock_write reports short writes as success

`modules/generators/mod_cgid.c:424` calls `write()` on the cgid Unix-domain
socket once and returns `APR_SUCCESS` for any non-negative result. Short
writes leave the daemon with a truncated request while the worker thinks
delivery completed.

## Setup

Stream-mode `AF_UNIX` writes can complete short when the kernel buffer
fills mid-call. Reproducing that against the real cgid daemon requires
tight timing, so this PoC isolates `sock_write()` into a tiny harness
that talks to a mock cgid daemon. A second variant patches an
`LD_PRELOAD` shim that returns a short `write()` count and runs a real
CGI request through `mod_cgid`.

Files:

- `tmp/sock_write_harness.c` - copy of the affected `sock_write()`.
- `tmp/short_write_preload.c` - shim that forces a short first write.
- `tmp/cgi-bin/dummy.cgi` - any executable script.

```bash
mkdir -p tmp/cgi-bin
printf '#!/bin/sh\necho "Content-Type: text/plain"; echo; echo ok\n' \
  > tmp/cgi-bin/dummy.cgi
chmod +x tmp/cgi-bin/dummy.cgi
```

## Trigger

Harness reproduction (kernel-driven short write on a tiny socket buffer):

```c
/* tmp/sock_write_harness.c */
#include <sys/socket.h>
#include <sys/un.h>
#include <unistd.h>
#include <stdio.h>
#include <string.h>

static int sock_write(int fd, const void *buf, size_t n) {
    int rc;
    do { rc = write(fd, buf, n); } while (rc < 0 && errno == EINTR);
    return rc < 0 ? -1 : 0;       /* mirrors the buggy helper */
}

int main(void) {
    int sv[2]; socketpair(AF_UNIX, SOCK_STREAM, 0, sv);
    int sndbuf = 4096;
    setsockopt(sv[0], SOL_SOCKET, SO_SNDBUF, &sndbuf, sizeof(sndbuf));
    char big[1<<20]; memset(big, 'A', sizeof(big));
    /* Drain side intentionally idle; first write returns SNDBUF bytes. */
    int rc = sock_write(sv[0], big, sizeof(big));
    int got = recv(sv[1], big, sizeof(big), MSG_DONTWAIT);
    printf("rc=%d delivered=%d of %zu\n", rc, got, sizeof(big));
}
```

```bash
cc -O0 -g tmp/sock_write_harness.c -o tmp/sock_write_harness
./tmp/sock_write_harness
```

End-to-end through httpd, forcing the short write with a preload shim:

```c
/* tmp/short_write_preload.c */
#define _GNU_SOURCE
#include <dlfcn.h>
#include <unistd.h>
static ssize_t (*real_write)(int, const void *, size_t);
ssize_t write(int fd, const void *b, size_t n) {
    if (!real_write) real_write = dlsym(RTLD_NEXT, "write");
    return real_write(fd, b, n > 16 ? 16 : n);   /* truncate */
}
```

```bash
cc -shared -fPIC tmp/short_write_preload.c -ldl -o tmp/shortw.so
LD_PRELOAD=$PWD/tmp/shortw.so ./httpd -X -f $PWD/tmp/httpd.conf &
curl -sS http://127.0.0.1:8080/cgi-bin/dummy.cgi
```

## Observed effect

Harness output:

```
rc=0 delivered=4096 of 1048576
```

`sock_write()` returned success, but only `SO_SNDBUF` bytes reached the
peer. With the preload, `mod_cgid` reports the request was sent while
the cgid daemon's `get_req()` either misparses the truncated header
(producing `Internal Server Error`/500) or blocks waiting for bytes
that never arrive, causing the request to hang until `Timeout` fires.
The error log shows `AH01259: error reading request from cgid daemon`
or a daemon-side parse warning.
