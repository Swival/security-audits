# PoC 052 — Windows listener thread handle leak

`create_listener_thread()` calls `_beginthreadex()` once per active listener
socket and discards the returned `HANDLE`. Each child process leaks one
kernel thread object reference per listener.

## Setup

Platform: Windows, WinNT MPM (`server/mpm/winnt/child.c`).

`httpd.conf` configured with multiple listeners so the leak is visible per
restart:

```
ServerName localhost
Listen 127.0.0.1:8080
Listen 127.0.0.1:8081
Listen 127.0.0.1:8082
Listen 127.0.0.1:8083
Listen 127.0.0.1:8084
Listen 127.0.0.1:8085
Listen 127.0.0.1:8086
Listen 127.0.0.1:8087

LoadModule mpm_winnt_module modules/mod_mpm_winnt.so
DocumentRoot "C:/Apache24/htdocs"
```

Eight listeners means eight `_beginthreadex()` calls per child startup.

## Trigger

Force repeated child recycles and watch the child's handle count climb:

```
httpd.exe -k start

for /L %i in (1,1,200) do (
    httpd.exe -k graceful
    timeout /t 1 /nobreak >NUL
)
```

Or PowerShell loop with sampling:

```
$child = Get-CimInstance Win32_Process -Filter "Name='httpd.exe' AND CommandLine LIKE '%-k start%'"
while ($true) {
    & httpd.exe -k graceful
    Start-Sleep -Seconds 1
    $h = (Get-Process -Id $child.ProcessId).HandleCount
    Write-Host "handles=$h"
}
```

## Observed effect

`HandleCount` for the child httpd.exe grows by N per `graceful` (where N is
the number of `Listen` directives), and never drops while the child stays
alive. Confirm via Process Explorer / SysInternals `handle.exe`:

```
handle.exe -p <child-pid> -s | findstr Thread
```

Pre-patch each iteration adds a `Thread` handle even though the listener
threads themselves complete and exit. Using `!handle 0 4 Thread` in WinDbg
on the child process shows accumulated zombie thread handles where
`ExitStatus != STILL_ACTIVE`.

Post-patch the count stabilises immediately after listener startup because
`CloseHandle()` releases the caller's reference once the thread is launched.

## Log marker

No log line is emitted for the leak itself. The detection signal is the
monotonically increasing handle count under `httpd.exe -k graceful` and
`!handle Thread` showing terminated thread objects still referenced.
