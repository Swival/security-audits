# 018 — Unchecked apr_finfo_t read in mod_cache_disk open_entity()

`open_entity()` at `modules/cache/mod_cache_disk.c:561` reads `finfo.inode` and
`finfo.device` even when `apr_file_info_get()` failed, exposing uninitialized
stack memory to the cache identity decision and to `dobj->file_size`.

## Setup

Build httpd with `mod_cache` and `mod_cache_disk`:

```bash
./configure --prefix=$PWD/inst \
    --enable-cache --enable-cache-disk --enable-headers
make -j && make install
```

`httpd.conf` snippet:

```apache
LoadModule cache_module modules/mod_cache.so
LoadModule cache_disk_module modules/mod_cache_disk.so

CacheRoot   "/tmp/httpd-cache"
CacheEnable disk "/cache/"
CacheDirLevels 2
CacheDirLength 1

DocumentRoot "/tmp/htdocs"
<Directory "/tmp/htdocs">
    Require all granted
</Directory>
Alias /cache/ /tmp/htdocs/
```

Populate a small object so a body file appears under `/tmp/httpd-cache/`:

```bash
mkdir -p /tmp/htdocs /tmp/httpd-cache
printf 'cached body\n' > /tmp/htdocs/obj.txt
curl -s -o /dev/null http://127.0.0.1:8080/cache/obj.txt
```

## Trigger

The bug requires `apr_file_info_get()` to fail on the already-opened body fd.
Two reproducible ways:

1. Static instrumentation. Apply this one-line patch on the unfixed source so the
   failure path is taken on every open, then drive a request:

```diff
@@ open_entity()
-        rc = apr_file_info_get(&finfo, APR_FINFO_SIZE | APR_FINFO_IDENT,
-                dobj->data.fd);
+        rc = APR_EBADF;   /* PoC injection */
```

```bash
curl -v http://127.0.0.1:8080/cache/obj.txt
```

2. Filesystem injection. Mount the cache directory on a backend that returns
   `EIO` for `fstat`-class calls (NBD with `nbdkit error`, or FUSE shim):

```bash
nbdkit -fU - eval get_size='echo 4096' \
       pread='dd if=/dev/zero bs=1 count=$3' \
       trim='exit 1'
mount -t nbd /tmp/httpd-cache
```

## Observed effect

Run httpd under MSan or Valgrind and replay the request:

```bash
valgrind --track-origins=yes -q ./inst/bin/httpd -X -f conf/httpd.conf &
curl -s http://127.0.0.1:8080/cache/obj.txt
```

Valgrind reports:

```text
==pid== Conditional jump or move depends on uninitialised value(s)
==pid==    at open_entity (mod_cache_disk.c:565)
==pid==  Uninitialised value was created by a stack allocation
==pid==    at open_entity (mod_cache_disk.c:540)
```

If the indeterminate `finfo.inode` happens to compare equal to `disk_info.inode`,
`dobj->file_size` is left at its prior value and a later `recall_body()` at
`mod_cache_disk.c:906` serves a truncated or oversized body. Otherwise the
entry is silently declined despite a valid cache file existing.

## Notes

The fix returns `DECLINED` on `apr_file_info_get()` failure before any
`finfo` field is read; the patched build no longer trips the Valgrind warning.
