# PoC 050 — uninitialized OCSP hook outputs in `stapling_cb`

`stapling_cb()` declares `ocsp_resp resp` and `int rspderlen` without
initialisation and passes their addresses to the
`get_stapling_status` hook. A hook that returns `APR_SUCCESS` without
populating both outputs makes Apache feed indeterminate stack data to
`SSL_set_tlsext_status_ocsp_resp()`.

## Setup

Build with `mod_ssl` plus a tiny test module that registers a faulty
`get_stapling_status` provider. Save as `mod_bad_stapling.c`:

```c
#include "httpd.h"
#include "http_config.h"
#include "mod_ssl.h"
#include "mod_ssl_openssl.h"

static apr_status_t bad_status(unsigned char **pder, int *pderlen,
                               conn_rec *c, server_rec *s, X509 *x)
{
    (void)pder; (void)pderlen; (void)c; (void)s; (void)x;
    return APR_SUCCESS;
}

static void register_hooks(apr_pool_t *p) {
    APR_OPTIONAL_HOOK(ssl, get_stapling_status, bad_status, NULL, NULL,
                      APR_HOOK_FIRST);
}
AP_DECLARE_MODULE(bad_stapling) = {
    STANDARD20_MODULE_STUFF, NULL, NULL, NULL, NULL, NULL, register_hooks
};
```

Compile and configure:

```
apxs -c -i mod_bad_stapling.c
```

```
Listen 127.0.0.1:443
LoadModule ssl_module          modules/mod_ssl.so
LoadModule bad_stapling_module modules/mod_bad_stapling.so
ServerName poc.local
SSLEngine on
SSLUseStapling on
SSLStaplingCache shmcb:logs/stapling(32768)
SSLCertificateFile    server.crt
SSLCertificateKeyFile server.key
```

Run httpd under ASan to catch indeterminate reads:

```
CFLAGS='-fsanitize=address -O1 -g' ./configure ... && make
ASAN_OPTIONS=detect_stack_use_after_return=1 httpd -X
```

## Trigger

Request a TLS handshake with the certificate-status extension:

```
printf '' | openssl s_client -connect 127.0.0.1:443 \
    -servername poc.local -status -msg 2>&1 | head -60
```

## Observed effect

Pre-patch (`resp.data` and `rspderlen` remain whatever the stack held):

- Best case: garbage bytes are stapled; `s_client` prints
  `OCSP response: no response data` after a failed parse.
- Worst case under ASan: `use-of-uninitialized-value` or a SEGV inside
  OpenSSL's `OCSP_response_create` ownership path.

Reproducible log marker:

```
[ssl:debug] AH01951 stapling_cb: OCSP Stapling callback called
==123==WARNING: MemorySanitizer: use-of-uninitialized-value
    #0 ... in stapling_cb modules/ssl/ssl_util_stapling.c:811
```

Post-patch, the same trigger logs `AH01959`-style "no OCSP response
provided" and returns `SSL_TLSEXT_ERR_NOACK` cleanly.
