# PoC 152 — Capacity-One FIFO Misses `not_full` Signal

For a capacity-one `h2_proxy_fifo`, `pull_head()` never broadcasts
`not_full`, so a producer blocked on a full FIFO is never woken when the
sole element is consumed.

## Setup

Same build harness as PoC 151 (link against `h2_proxy_util.c` and APR).

`poc152.c`:

```c
#include <stdio.h>
#include <apr_thread_proc.h>
#include "h2_proxy_util.h"

static h2_proxy_fifo *f;

static void *APR_THREAD_FUNC producer(apr_thread_t *t, void *arg)
{
    apr_status_t rv = h2_proxy_fifo_push(f, "B");   /* blocks; FIFO full */
    fprintf(stderr, "second push returned %d\n", rv);
    apr_thread_exit(t, 0); return NULL;
}

int main(void)
{
    apr_pool_t *p; apr_thread_t *th;
    apr_initialize(); apr_pool_create(&p, NULL);
    h2_proxy_fifo_create(&f, p, 1);                 /* capacity = 1 */
    h2_proxy_fifo_push(f, "A");                     /* fills FIFO   */
    apr_thread_create(&th, NULL, producer, NULL, p);
    apr_sleep(apr_time_from_sec(1));                /* producer parked */

    void *e;
    h2_proxy_fifo_pull(f, &e);                      /* count: 1 -> 0  */
    fprintf(stderr, "pulled %s; producer should now wake\n", (char*)e);

    apr_status_t rv;
    apr_thread_join(&rv, th);                       /* hangs           */
    return 0;
}
```

## Trigger

```bash
make poc152
timeout 5 ./poc152 ; echo "exit=$?"
```

## Observed effect

* Output prints `pulled A; producer should now wake`.
* The second-push line never appears; the join blocks indefinitely.
* `timeout` ends with `exit=124`.
* A control run with capacity 2 — push twice, pull once — wakes the
  producer immediately, demonstrating the bug is specific to the
  `1 -> 0` transition skipped by the `if (fifo->count > 0)` branch at
  `modules/http2/h2_proxy_util.c:1270`.
