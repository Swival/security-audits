# PoC 159 — Wrong Certificate Index During `md_reg_freeze_domains()`

`md_reg_freeze_domains()` preloads pubcerts using the outer loop index
`i` instead of the inner certificate index `j`. Later certificate
indexes are never cached, so `md_reg_get_pubcert(md, j>0)` returns
`APR_ENOENT` after freeze even though the certificate is on disk.

## Setup

`mod_md` with a managed domain that has both EC and RSA key specs (two
certificate indexes per MD).

```bash
mkdir -p /tmp/httpd-poc-159/md
cd /tmp/httpd-poc-159
```

`httpd.conf`:

```apache
ServerRoot "/tmp/httpd-poc-159"
PidFile "/tmp/httpd-poc-159/httpd.pid"
ErrorLog "/tmp/httpd-poc-159/error.log"
LogLevel md:trace4 ssl:info

Listen 127.0.0.1:8443

LoadModule mpm_event_module    modules/mod_mpm_event.so
LoadModule authz_core_module   modules/mod_authz_core.so
LoadModule ssl_module          modules/mod_ssl.so
LoadModule watchdog_module     modules/mod_watchdog.so
LoadModule md_module           modules/mod_md.so

MDStoreDir /tmp/httpd-poc-159/md
MDPrivateKeys secp256r1 RSA 2048
MDomain example.test

<VirtualHost 127.0.0.1:8443>
    ServerName example.test
    SSLEngine on
</VirtualHost>
```

Pre-populate the store so that index 0 (EC) is missing and index 1
(RSA) is present. This mirrors the reproduced trigger in the finding
where `state_init()` caches the missing earlier index and stops:

```bash
# Layout under MDStoreDir/domains/example.test/
mkdir -p md/domains/example.test
# Place a valid RSA pubcert at index 1's expected filename only.
openssl req -x509 -newkey rsa:2048 -nodes -days 1 \
  -subj "/CN=example.test" \
  -keyout md/domains/example.test/privkey.rsa.pem \
  -out    md/domains/example.test/pubcert.rsa.pem
# Intentionally do NOT create pubcert.secp256r1.pem.
```

## Trigger

```bash
/usr/sbin/httpd -f /tmp/httpd-poc-159/httpd.conf -X &
HTTPD=$!
sleep 2
# After post-config, mod_md calls md_reg_freeze_domains().
# The status hook will then ask for cert index 1 (RSA).
curl -sk https://127.0.0.1:8443/server-status?auto
kill $HTTPD
grep -E 'pubcert|freeze|ENOENT' /tmp/httpd-poc-159/error.log
```

## Observed effect

`error.log` shows the freeze loop calling
`md_reg_get_pubcert(..., i=0, ...)` twice (once per inner-loop iteration)
and never with `j=1`:

```
md_reg_freeze_domains: preload md=example.test idx=0
md_reg_freeze_domains: preload md=example.test idx=0
```

Subsequent renewal/status callers that legitimately request index 1
hit:

```
md_reg_get_pubcert(example.test, idx=1) -> APR_ENOENT (frozen)
```

Even though `pubcert.rsa.pem` exists on disk. With the patch
(`md_reg_get_pubcert(&pubcert, reg, md, j, reg->p)`), the freeze loop
preloads both indexes and the same call returns `APR_SUCCESS`,
confirming the wrong-index bug at `modules/md/md_reg.c:1109`.
