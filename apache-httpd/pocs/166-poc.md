# PoC 166 — Unchecked crypt() Result Reaches strcmp() (mod_lua passwd)

`mk_password_hash()` passes the return value of a second `crypt()` call
straight into `strcmp()` without checking for `NULL`, so a `crypt()` failure on
the truncation-check path becomes a `NULL`-pointer dereference inside the Lua
worker.

## Setup

Module: `mod_lua`. The vulnerable function is reached via
`r:htpassword(password, ALG_CRYPT)` where `password` is longer than 8 bytes.

`/tmp/166/htdocs/login.lua`:

```lua
function handle(r)
    r.content_type = "text/plain"
    local pw = r:parsebody()["pw"]
    if pw then
        local h = r:htpassword(pw, 1)  -- ALG_CRYPT == 1
        r:puts(tostring(h))
    end
    return apache2.OK
end
```

`/tmp/166/httpd.conf` (essentials):

```apache
ServerRoot /tmp/166
Listen 127.0.0.1:8166
LoadModule lua_module        modules/mod_lua.so
LoadModule mpm_event_module  modules/mod_mpm_event.so
LoadModule unixd_module      modules/mod_unixd.so
LoadModule authz_core_module modules/mod_authz_core.so
DocumentRoot /tmp/166/htdocs
ErrorLog /tmp/166/error.log
<Files "login.lua">
    SetHandler lua-script
</Files>
```

## Trigger

`crypt()` failures depend on the platform `libcrypt`. On glibc with libxcrypt
the easiest way to make `crypt()` return `NULL` is to pass a salt the backend
rejects. Since `mk_password_hash()` chooses the salt internally, the lab
forces failure with an `LD_PRELOAD` shim:

`/tmp/166/cryptshim.c`:

```c
#define _GNU_SOURCE
#include <string.h>
#include <errno.h>
#include <stddef.h>
static int n;
char *crypt(const char *key, const char *salt) {
    if (++n >= 2) { errno = EINVAL; return NULL; }   /* fail second call */
    static char out[14];
    out[0] = salt[0]; out[1] = salt[1];
    for (int i = 2; i < 13; i++) out[i] = 'A' + (key[i % strlen(key)] & 31);
    out[13] = 0;
    return out;
}
```

```sh
cc -fPIC -shared /tmp/166/cryptshim.c -o /tmp/166/cryptshim.so
LD_PRELOAD=/tmp/166/cryptshim.so httpd -X -f /tmp/166/httpd.conf &
curl --data 'pw=correcthorsebatterystaple' http://127.0.0.1:8166/login.lua
```

## Observed effect

- The single `-X` worker crashes with `SIGSEGV` inside
  `mk_password_hash()` at `modules/lua/lua_passwd.c:138`. Stack trace
  (compiled with `-g`):

  ```text
  #0 __strcmp_avx2 ()
  #1 mk_password_hash (ctx=...) at modules/lua/lua_passwd.c:138
  #2 lua_apr_b64encode/htpassword glue
  ```

- `error.log` records the abnormal exit; with system core dumps enabled the
  faulting instruction is the first `movzbl (%rsi), %eax` of `strcmp` with
  `%rsi == 0`.

- `curl` returns `52 Empty reply from server` and the connection is dropped.

After the patch (`cbuf = crypt(...); if (cbuf != NULL && !strcmp(...))`) the
warning branch is simply skipped on `crypt()` failure and the request returns
the primary hash without crashing.

## Cleanup

```sh
kill %1 2>/dev/null
rm -f /tmp/166/cryptshim.so /tmp/166/error.log
```
