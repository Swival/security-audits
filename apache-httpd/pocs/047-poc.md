# PoC 047 — HTTP/2 mplx aborted/shutdown flag race

`workers_shutdown()` writes `m->shutdown` and `m->aborted` under `m->poll_lock`
while c1/session readers consult those fields under `m->lock`. Concurrent child
shutdown plus active stream processing produces a non-atomic data race.

## Setup

Build httpd with HTTP/2 enabled and ThreadSanitizer or Helgrind:

```
CFLAGS='-fsanitize=thread -O1 -g' LDFLAGS='-fsanitize=thread' \
  ./configure --enable-http2 --enable-ssl \
              --with-mpm=event --enable-mods-static=few
make -j4
```

Minimum `httpd.conf`:

```
Listen 127.0.0.1:8443
LoadModule ssl_module    modules/mod_ssl.so
LoadModule http2_module  modules/mod_http2.so
ServerName localhost
SSLEngine on
SSLCertificateFile    server.crt
SSLCertificateKeyFile server.key
Protocols h2 http/1.1
H2MinWorkers 2
H2MaxWorkers 4
MaxConnectionsPerChild 8
DocumentRoot /var/www
```

Generate certs:

```
openssl req -x509 -newkey rsa:2048 -nodes -keyout server.key \
  -out server.crt -subj '/CN=localhost' -days 30
```

## Trigger

Drive parallel HTTP/2 traffic so workers and session threads contend, then
issue a graceful child recycle so `workers_shutdown()` fires while c1 paths
are still polling/reprioritising:

```
# Terminal A — sustained h2 load to saturate worker pool
for i in $(seq 1 200); do
  h2load -n 200 -c 8 -m 16 https://127.0.0.1:8443/ &
done
wait

# Terminal B — repeatedly trip MaxConnectionsPerChild recycle
while :; do
  apachectl -k graceful
  sleep 0.1
done
```

## Observed effect

ThreadSanitizer logs a `WARNING: ThreadSanitizer: data race` against
`modules/http2/h2_mplx.c:1053` (writer) versus reads in
`h2_mplx_c1_poll`, `h2_mplx_c1_reprioritize`, `c1_process_stream`,
`c2_prod_next`, and `mplx_pollset_poll` (`h2_mplx.c:~870`). Without TSan,
intermittent symptoms include sessions continuing to schedule streams after
`m->aborted = 1`, and `mplx_pollset_poll()` interpreting an ungraceful
shutdown as a graceful wakeup.

Stress driver to amplify timing:

```
python3 -c '
import os, subprocess, time
for _ in range(50):
    subprocess.Popen(["h2load","-n","100","-c","4","-m","8",
                      "https://127.0.0.1:8443/"])
time.sleep(1)
os.system("apachectl -k graceful")
'
```

[tsan] data race in h2_mplx.c:1053 (workers_shutdown) vs h2_mplx.c:870
