# 040 - mod_dav_lock indirect-record key length not validated

`modules/dav/lock/locks.c:554` reads `ip->key.dsize` from a DBM record
and uses it directly in `apr_pmemdup()` without verifying that many
bytes remain. A crafted indirect record can read past the SDBM
allocation.

## Setup

Same DAV configuration as PoC 038/039. Indirect records are produced
when a lock is created on a parent collection with `Depth: infinity`,
which leaves indirect entries on every child resource.
