# PoC 049 — `set_challenge_creds()` reports failures as success

`set_challenge_creds()` discards `rv` at the `cleanup` label and always
returns `APR_SUCCESS`, so ALPN selection never sends a fatal alert when an
ACME TLS-ALPN-01 challenge cert/key fails to install.

## Setup

Build with `mod_ssl` and `mod_md`:

```
./configure --enable-ssl --enable-md --enable-http2 \
            --with-mpm=event --enable-mods-static=few
make -j4
```

Minimum `httpd.conf`:

```
Listen 127.0.0.1:443
LoadModule ssl_module   modules/mod_ssl.so
LoadModule md_module    modules/mod_md.so
LoadModule http2_module modules/mod_http2.so

ServerName poc.local
MDomain poc.local
MDCertificateAuthority http://127.0.0.1:14000/dir
MDChallenges tls-alpn-01

<VirtualHost *:443>
    ServerName poc.local
    SSLEngine on
    SSLCertificateFile    fallback.crt
    SSLCertificateKeyFile fallback.key
    Protocols acme-tls/1 h2 http/1.1
</VirtualHost>
```

Drop a deliberately corrupt challenge cert into `mod_md`'s challenge store
so `modssl_read_cert()` parse fails:

```
mkdir -p md/challenges/poc.local
printf -- '-----BEGIN CERTIFICATE-----\nNOTREALLYPEM\n-----END CERTIFICATE-----\n' \
  > md/challenges/poc.local/acme-tls-alpn-01.cert.pem
openssl genrsa 2048 > md/challenges/poc.local/acme-tls-alpn-01.key.pem
```

## Trigger

Speak `acme-tls/1` ALPN with SNI `poc.local`:

```
printf '' | openssl s_client -connect 127.0.0.1:443 \
   -servername poc.local -alpn acme-tls/1 -tls1_2 -msg 2>&1 | head -50
```

## Observed effect

Without the patch the handshake completes against the fallback certificate
instead of failing. Server `error_log`:

```
AH10266 Failed to load challenge certificate ...
```

but the client sees `SSL_connect` succeed and the negotiated protocol is
`acme-tls/1` without the proper challenge identity. Wireshark/`-msg`
confirms no fatal alert is sent after the failed PEM parse.

With the patch, the same trigger yields:

```
<<< TLS 1.2 Alert (fatal, internal_error/handshake_failure)
```

and `s_client` reports `SSL_connect:error in error`. ACME validation
succeeds (incorrectly) before the patch and fails (correctly) after.
