# PoC 140 - Stale match state across providers in mod_filter

`filter_lookup()` reuses `match` across iterations. When provider A matches
its dispatch condition but is dropped by `proxy=no`, the next provider
inherits `match == 1` and runs even when its own condition is false.

## Setup

Lab httpd 2.4.66 with `mod_filter`, `mod_proxy`, `mod_proxy_http`,
`mod_substitute`, and `mod_deflate`. The smart filter has two providers,
the first guarded by an always-true expression but with `proxy=no`, the
second guarded by `Content-Type=text/plain`.

`conf/httpd.conf`:

```
ServerName localhost
Listen 127.0.0.1:8080
LoadModule mpm_event_module        modules/mod_mpm_event.so
LoadModule authz_core_module       modules/mod_authz_core.so
LoadModule filter_module           modules/mod_filter.so
LoadModule substitute_module       modules/mod_substitute.so
LoadModule deflate_module          modules/mod_deflate.so
LoadModule proxy_module            modules/mod_proxy.so
LoadModule proxy_http_module       modules/mod_proxy_http.so
LoadModule unixd_module            modules/mod_unixd.so
LoadModule log_config_module       modules/mod_log_config.so

ErrorLog logs/error_log
LogLevel filter:trace5

ProxyPass        /up http://127.0.0.1:9000
ProxyPassReverse /up http://127.0.0.1:9000

FilterDeclare    SMARTF
FilterProvider   SMARTF SUBSTITUTE  "%{REQUEST_METHOD} = 'GET'"
FilterProvider   SMARTF DEFLATE     "%{CONTENT_TYPE} =~ m{^text/plain}"
FilterProtocol   SMARTF SUBSTITUTE  "proxy=no"
FilterChain      +SMARTF

Substitute       "s/HELLO/PWNED/n"
```

Origin `127.0.0.1:9000` (a tiny Python upstream that sets
`Content-Type: text/html` and never matches the `text/plain` provider):

```
python3 - <<'PY'
import http.server
class H(http.server.BaseHTTPRequestHandler):
    def do_GET(self):
        body = b"HELLO from origin\n"
        self.send_response(200); self.send_header("Content-Type","text/html")
        self.send_header("Content-Length", str(len(body))); self.end_headers()
        self.wfile.write(body)
http.server.HTTPServer(("127.0.0.1",9000), H).serve_forever()
PY
```

## Trigger

```
curl -v --compressed http://127.0.0.1:8080/up/
```

## Observed effect

- The proxy response carries `Content-Type: text/html`.
- Provider `SUBSTITUTE` matches the `REQUEST_METHOD` expression but is
  dropped because of `proxy=no`. `match` is still `1`.
- Provider `DEFLATE` does not match `text/plain`, but stale `match == 1`
  selects it. The response is gzipped despite the `Content-Type` rule.
- `LogLevel filter:trace5` shows `Content-Type 'text/html' did not match,
  but provider DEFLATE selected anyway`.
- Curl reports `Content-Encoding: gzip` and `--compressed` decodes successfully.

Switch the upstream to return `text/plain` with `HELLO`: provider
`SUBSTITUTE` is still dropped (proxy), and `DEFLATE` legitimately matches.
That produces the same wire effect, masking the bug. Use the `text/html`
case to demonstrate the stale state.

After applying the patch (`match = 0` at the top of every provider loop
iteration), the `text/html` response is delivered uncompressed.
