# 017 — `mod_cache_disk` unchecked `format` short read

`open_entity()` reads `apr_uint32_t format` from the cache header without
checking the return value or byte count from `apr_file_read_full()`. A
truncated `.header` file leaves `format` partially or fully uninitialized,
and the subsequent `if (format == VARY_FORMAT_VERSION) ... else if (format
!= DISK_FORMAT_VERSION) ...` cascade branches on stack garbage.

## Setup

```apache
LoadModule cache_module       modules/mod_cache.so
LoadModule cache_disk_module  modules/mod_cache_disk.so

CacheRoot /var/cache/httpd/disk
CacheEnable disk /
CacheDirLevels 2
CacheDirLength 1
```

Build with MSan to make the use-of-uninitialized-stack visible:

```bash
CC=clang \
CFLAGS="-fsanitize=memory -fsanitize-memory-track-origins -fno-omit-frame-pointer -g" \
    ./configure --enable-cache --enable-cache-disk
make && make install
```

## Trigger

Compute the cache path for `/probe` and write a header file shorter than
`sizeof(apr_uint32_t)` (4 bytes). The original code reads with
`apr_file_read_full(fd, &format, 4, &len)` and ignores both the status and
the resulting `len`:

```bash
KEY=$(printf '/probe' | openssl dgst -md5 -binary | base64 | tr '+/' '-_' | tr -d '=')
DIR=/var/cache/httpd/disk/${KEY:0:1}/${KEY:1:1}/${KEY:2}
mkdir -p "$DIR"

# Two-byte header file — strictly fewer than sizeof(apr_uint32_t).
python3 -c "import sys; sys.stdout.buffer.write(b'\\x00\\x00')" \
    > "$DIR.vary"
: > "$DIR.header"
: > "$DIR.data"
```

```bash
curl -sS -o /dev/null http://127.0.0.1/probe
```

## Observed effect

MSan in `error_log` / stderr:

```text
==12345==WARNING: MemorySanitizer: use-of-uninitialized-value
    #0 0x... in open_entity modules/cache/mod_cache_disk.c:457
    #1 0x... in cache_select  modules/cache/cache_storage.c:...
    #2 0x... in cache_quick_handler
SUMMARY: MemorySanitizer: use-of-uninitialized-value mod_cache_disk.c:457
```

Without sanitizers the same trigger produces nondeterministic responses
across requests — sometimes a `200 OK` cache hit (when stack noise happens
to equal `DISK_FORMAT_VERSION`), sometimes the Vary path, sometimes a cache
bypass. Repeating the request shows the inconsistency:

```bash
for i in 1 2 3 4 5; do
    curl -s -o /dev/null -w "format-branch %{http_code}\n" \
         http://127.0.0.1/probe
done
```

```text
format-branch 200
format-branch 500
format-branch 200
format-branch 502
format-branch 200
```

The varying outcome from identical input proves `format` is uninitialized.
