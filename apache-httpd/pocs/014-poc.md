# 014 — Event MPM pollset size 32-bit wrap

`setup_threads_runtime()` computes the pollset capacity as
`num_listensocks + threads_per_child * (async_factor > 2 ? async_factor : 2)`
in `apr_uint32_t`. A large `AsyncRequestWorkerFactor` makes the
multiplication wrap, producing a tiny pollset for a high-concurrency
configuration and triggering connection-shedding once async slots fill.

## Setup

```apache
LoadModule mpm_event_module modules/mod_mpm_event.so

ServerLimit         1
StartServers        1
ThreadsPerChild     25
MaxRequestWorkers   25

AsyncRequestWorkerFactor 171798692
```

The chosen `AsyncRequestWorkerFactor` value matches the finding's
`async_factor=171798692` (`worker_factor = 2748779072`). With
`threads_per_child = 25`:

`25 * 171798692 = 4294967300` which wraps to `4` modulo `2**32`. Adding the
listener slot and the wakeable pipe slot still gives a pollset of size 5 or
6 — far below the 25 keepalive/async slots the configuration intends.

## Trigger

Start httpd, then open more concurrent keep-alive connections than the
wrapped pollset can hold:

```bash
httpd -X &
sleep 1

python3 - <<'PY'
import socket, threading, time
def hold(i):
    s = socket.create_connection(("127.0.0.1", 80))
    s.sendall(b"GET / HTTP/1.1\r\nHost: x\r\nConnection: keep-alive\r\n\r\n")
    s.recv(4096)
    time.sleep(60)        # park the connection in keepalive
for i in range(40):
    threading.Thread(target=hold, args=(i,), daemon=True).start()
time.sleep(120)
PY
```

## Observed effect

`error_log` shows pollset failures and graceful-restart signaling:

```text
[mpm_event:error] AH00485: failed to add to pollset (Argument out of range)
[mpm_event:error] AH02061: Too many open polled sockets
[mpm_event:warn]  child process is graceful-restart signaled
```

gdb at `setup_threads_runtime()` confirms the wrap directly:

```text
(gdb) p async_factor
$1 = 171798692
(gdb) p threads_per_child
$2 = 25
(gdb) p (apr_uint32_t)num_listensocks + (apr_uint32_t)threads_per_child * \
        (async_factor > 2 ? async_factor : 2)
$3 = 5
(gdb) p pollset_size
$4 = 5
```

A connection load above 5 simultaneous polled sockets is enough to drive the
worker into graceful restart.
