# PoC 072 - Unsynchronized h2_ififo_count Read

`h2_ififo_count()` reads `fifo->count` without holding `fifo->lock`, racing with
mutex-protected writers in `h2_ififo_push/pull/remove`.

## Setup

This is a C-level data race in `modules/http2/h2_util.c:958`. Reproduction is
easiest with a minimal harness against the source. ThreadSanitizer (tsan) flags
the race directly.

`tmp/race.c`:

```c
#include "h2_util.h"
#include <pthread.h>

static h2_ififo *fifo;

static void *writer(void *arg) {
    for (int i = 0; i < 100000; i++) {
        h2_ififo_push(fifo, i & 0xffff);
        int v;
        h2_ififo_pull(fifo, &v);
    }
    return NULL;
}

static void *reader(void *arg) {
    volatile int n;
    for (int i = 0; i < 100000; i++) n = h2_ififo_count(fifo);
    return NULL;
}

int main(void) {
    apr_initialize();
    apr_pool_t *p; apr_pool_create(&p, NULL);
    h2_ififo_create(&fifo, p, 4096);
    pthread_t t1, t2;
    pthread_create(&t1, NULL, writer, NULL);
    pthread_create(&t2, NULL, reader, NULL);
    pthread_join(t1, NULL); pthread_join(t2, NULL);
    return 0;
}
```

httpd.conf (live demonstration with HTTP/2 under load):

```apache
LoadModule http2_module modules/mod_http2.so
Listen 443
Protocols h2 http/1.1
SSLEngine on
SSLCertificateFile /tmp/cert.pem
SSLCertificateKeyFile /tmp/key.pem
DocumentRoot /srv/www
```

## Trigger

Compile httpd with `-fsanitize=thread` and drive concurrent HTTP/2 requests
across many streams sharing one connection:

```bash
h2load -n 200000 -c 32 -m 100 https://localhost/index.html
```

Equivalent harness build:

```bash
cc -fsanitize=thread -I include -I srclib/apr/include \
   tmp/race.c modules/http2/h2_util.c -lapr-1 -lpthread -o tmp/race
./tmp/race
```

## Observed effect

ThreadSanitizer prints:

```text
WARNING: ThreadSanitizer: data race
  Read of size 4 at 0x... by thread T2:
    #0 h2_ififo_count modules/http2/h2_util.c:958
  Previous write of size 4 at 0x... by thread T1:
    #0 ififo_push_int modules/http2/h2_util.c (under fifo->lock)
```

Without tsan the race is silent; observed values returned by
`h2_ififo_count()` may be momentarily out of range (negative or higher than the
queue capacity) on weakly ordered architectures, leading to incorrect flow
control accounting in higher layers.

## Log marker

ThreadSanitizer report only; httpd error log does not record the race. Confirm
the call site by setting a breakpoint at `h2_util.c:958` under gdb and noting
`fifo->lock` is not held.
